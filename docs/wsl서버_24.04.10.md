# WSL 서버 작업 정리 (2026-04-10)

## 1. 작업 개요
- 카카오톡 전송 로직, 협정보드 점수 계산/표시, 의무지역 모달 표시값, 협정보드 편의기능(업체정보갱신) 관련 요청을 반영함.
- 특히 LH 구간의 실적점수 계산/표시 규칙을 재정리하고, 점수 뻥튀기 이슈 원인을 정리 후 수정함.

## 2. 카카오톡 전송(팀장님 제외) 관련
- 팀장님 제외 로직에서 `담당자 제외`와 별도로 `업체 제외` 목록을 추가 관리하도록 정리함.
- 추가 요청 업체:
  - 아람이엔테크㈜
  - ㈜우진일렉트
  - 에코엠이엔씨㈜
  - (주)지음쏠라테크
- 협정 문자의 팀장님 제외 판정은 다음 기준으로 확인:
  - 협정문에 포함된 업체가 **전부 제외 목록**일 때만 팀장님에서 제외
  - 포함 업체 중 **1개라도 제외 목록이 아니면** 팀장님 목록에 포함

## 3. 복사 버튼 이슈
- 개발서버 테스트 중 `복사에 실패했습니다. 텍스트를 직접 선택해 주세요.` 메시지 확인.
- 브라우저 보안 정책(클립보드 권한/포커스/보안 컨텍스트) 영향 가능성을 안내하고, 운영 환경과 분리해서 점검하도록 정리.

## 4. 의무지역 모달 표시 버그
- 의무지역 선택 모달에서 대표지역 외 값(예: `경상북도 포항시`, `전북 남원시`)이 노출되던 문제 수정.
- 대표지역 단위만 보이도록 정리.
- 기존 데이터 시트명(`경북` 등)을 중복 생성하거나 변경하는 방식이 아니라, 표시/정규화 단계 수정으로 대응.

## 5. 협정보드 "업체정보갱신" 기능
- 협정보드에 `업체정보갱신` 버튼 추가.
- 위치: `문자전송완료` 버튼 옆.
- 동작: 현재 협정보드에 배치된 업체의 최신 데이터(예: 갱신된 신용평가/점수 항목)를 다시 반영.
- 헤더 정보(공고번호/공고명/금액/발주처/기타 설정)는 유지한 상태로 업체 데이터만 갱신.

## 6. 예상점수/실적점수 이슈 정리

### A. 점수 뻥튀기 원인
- `raw`(비캡핑 원점수) 값을 총점 경로에 섞어 쓰면서 95점 초과 등 비정상 표시가 생길 수 있는 구조가 있었음.
- 총점 계산은 캡핑된 점수 기준으로 유지하고, raw는 참조/표시 용도로만 분리하는 방향으로 정리.

### B. 실적계수 표기 정책 조정
- 실적계수 별도 칸을 전 발주처에 확장했다가, 화면 공간 이슈로 정책 변경.
- 최종:
  - `LH 100억~300억`만 실적계수 별도 표시 유지
  - 그 외 구간(요청된 LH/조달청/도로공사)은 별도 칸 제거, 실적총점에 점수값 표시

### C. LH 실적계수/실적점수 규칙 확정
- LH 50억미만, LH 50억~100억:
  - 실적계수 계산 후
  - `소수점 셋째 자리에서 반올림`(= 소수 둘째 자리까지)
  - 그 다음 `* 13`으로 실적총점 계산
- LH 100억~300억:
  - 기존 규칙 유지(별도 실적계수 표기 유지)
  - 실적계수 표시는 반올림 없이 소수 4자리 표시 기준 유지

### D. 조달청/도로공사 반올림 정책(현 상태 유지)
- 조달청/도로공사는 기존 구현 그대로 유지하기로 최종 합의.
- 현재 구현 기준:
  - 조달청: 실적총점 단계에서 소수 둘째 자리 올림
  - 도로공사: 실적총점 단계에서 소수 둘째 자리 반올림
