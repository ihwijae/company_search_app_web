## 2) 인증/계정 관련 반영

### 2-1. 계정 저장/생성
- 계정 저장 파일: `users.yaml` (JSON-compatible 형식)
- 커밋 제외: `.gitignore`에 `users.yaml` 추가
- 계정 생성 명령:
```bash
npm run auth:create-user
```
- 현재 스크립트는 인자 없이 실행 시 터미널 입력 순서:
  1. `Name`
  2. `ID`
  3. `Password`
  4. `Confirm Password`
- 비밀번호는 평문 저장하지 않고 `scrypt` 해시로 저장.

### 2-2. 인증 API/세션
- 추가 API:
  - `GET /api/auth?action=session`
  - `POST /api/auth?action=login`
  - `POST /api/auth?action=logout`
- 세션 쿠키 기반 인증 적용(`HttpOnly`, `SameSite=Lax`).
- 세션 TTL 기본값: `8시간` (`AUTH_SESSION_TTL_MS`로 변경 가능).
- `/api/auth` 제외 모든 `/api/*` 요청은 인증 필요(미인증 시 `401`).

### 2-3. 로그인 UI
- 로그인 페이지 추가 및 앱 진입 가드 적용.
- 미로그인 상태에서 앱 접근 시 로그인 페이지로 이동.

## 3) 사용자명 표시/로그아웃 UI
- 사용자명 표시를 상단 메뉴바 우측으로 이동.
- 사용자명 클릭 시 드롭다운에 `로그아웃` 버튼 노출.
- 외부 클릭 시 드롭다운 닫힘.
- 스크롤 영역 분리(`nav-scroll` + `nav-user`)로 드롭다운 잘림/오작동 문제 해결.

## 4) 협정보드 - 문자전송완료 구분 기능

### 4-1. 상태 저장 구조
- 협정 메타에 아래 필드 추가:
  - `smsStatus` (`pending` | `sent`)
  - `smsCompletedAt`
- 상태 변경 API 추가:
  - `POST /api/agreement-board` with `action: "set-sms-status"`

### 4-2. 협정보드 메인 버튼
- `문자전송완료` 버튼 추가(후보 버튼 옆 배치).
- 완료 상태면 `문자전송완료 해제`로 토글.
- 버튼 전용 색상 스타일 적용.

### 4-3. 초기화 시 동작 보정
- `초기화` 실행 시:
  - 화면의 문자전송완료 상태를 미완료로 리셋
  - 현재 협정 path 연결 해제
- 목적:
  - 다른 협정 상태가 잘못 해제되는 문제 방지
  - 협정별 상태 분리 유지

## 5) 협정 불러오기 모달 개선

### 5-1. 크기/레이아웃
- 모달 폭/높이 확대.
- 페이지네이션 버튼(`처음/이전/다음/끝`) 잘림 방지.

### 5-2. 문자전송 상태 표시
- 제목 우측 배지: `✓ 전송완료`
- 액션 영역 원형 아이콘:
  - 완료: `✓`
  - 미완료: `○`
- 필터 추가:
  - `전체`(기본)
  - `전송미완료`
  - `전송완료`

### 5-3. 작업자 배지
- 저장 메타에 `savedById`, `savedByName` 기록.
- 목록에서 `작업자 {이름}` 배지 표시.
- 동일 협정을 다른 사용자가 저장하면 마지막 저장자 기준으로 갱신.

## 6) 운영 실행 명령 정리

### 6-1. 서버 실행
- 외부 공개(터널만 사용):
```bash
cd ~/projects/company_search_app_web
npm run serve:prod
```

- 사내 LAN 직접 접속까지 필요할 때:
```bash
cd ~/projects/company_search_app_web
npm run serve:prod:lan
```

### 6-2. 터널 실행
```bash
cloudflared tunnel run company-search
```
