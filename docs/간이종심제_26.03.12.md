# 간이종심제 작업 메모 (2026-03-12)

## 목적

2026-03-12 세션에서 진행한 `LH 100억~300억 간이종심제` 점검, 문제 분석, 수정 내용을 다음 세션에서 바로 이어갈 수 있도록 정리한다.

이번 세션 핵심은 아래 4가지다.

- 간이종심제 멈춤 원인 분석
- 간이종심제 전용 흐름 제거
- LH 50억 미만 / 50억~100억 실적점수 엑셀 기준 정합
- 협정보드 부가 기능 보정

---

## 1. 결론 요약

이번 세션에서 확인된 핵심은 다음과 같다.

- 간이종심제가 멈춘 직접 원인은 `지역경제 기여도 점수 자체`가 아니라
  `간이종심제만 공통 group-summary 흐름과 다르게 타던 별도 계산 경로`였다.
- 현재는 간이종심제도 다른 발주처처럼 공통 `group-summary` 흐름을 타고,
  `기준값만 간이종심제 기준`으로 적용하는 구조로 정리했다.
- 즉 별도 계산 흐름은 제거했고, 현재는
  - 지역업체면 슬롯 기준값 `1.5`
  - 최종 지역경제 기여도 max `0.3`
  - 기타점수 `17`
  - 최종 만점 `40`
  같은 기준만 간이종심제 버전으로 다르게 들어간다.

## 2. 간이종심제 멈춤 문제 분석

### 증상

- 협정보드에서 `LH 100억~300억` 선택 시 즉시 화면이 멈춤
- 클릭이 안 되고, 앱 자체를 다시 켜야 복구되는 경우가 있었음
- 업체 제거/배치 과정에서도 같은 증상이 재현됨

### 잘못 짚었던 후보

처음에는 아래 후보들을 의심했다.

- 후보 점수 재평가(`candidate-eval`)
- 실적계수 / 지역경제 기여도 계수
- 품질점수
- 실적점수 계산식

하지만 단계적으로 확인한 결과, 핵심 원인은 이쪽이 아니었다.

### 실제 원인

실제 문제는 `group-summary` 안의 간이종심제 전용 계산 흐름이었다.

특히 문제였던 부분:

- 간이종심제만 `regional-share` 방식으로 별도 지역경제 기여도 계산
- 후보가 `지역업체인지`를 `group-summary` 안에서 별도로 판단
- 같은 결과여도 `setGroupSummaries(results)`가 계속 다시 실행될 수 있는 구조

즉 흐름은 아래처럼 꼬여 있었다.

1. 간이종심제 선택
2. `group-summary` 실행
3. 간이종심제 전용 지역경제 기여도 계산 경로 실행
4. 결과를 `setGroupSummaries(results)`로 저장
5. 다시 렌더
6. 다시 `group-summary`

이 반복으로 UI가 멈췄다.

## 3. 이번 세션에서 실제로 수정한 간이종심제 구조

### 3-1. 간이종심제 전용 흐름 제거

수정 파일:

- `src/view/features/agreements/components/AgreementBoardWindow.jsx`
- `src/shared/agreements/calculations/groupSummary.js`

정리한 내용:

- 간이종심제만 따로 타던 `regional-share` 경로 제거
- 공통 `group-summary` 흐름 그대로 사용
- `group-summary` 안에서 간이종심제만 후보의 `지역업체 여부`를 따로 읽어 흐름을 바꾸던 코드 제거
- `group-summary` 실적 계산 호출에 간이종심제 전용 `perfCoefficient` 전달 제거

현재 의미:

- 다른 발주처처럼 공통 흐름 사용
- 간이종심제는 계산 기준만 다름

### 3-2. 지역경제 기여도 계산 복구 방식

간이종심제는 `신인도`가 아니라 `지역경제 기여도`를 사용한다.

현재 구조:

- 공통 `manual` 가중합 흐름 사용
- 간이종심제에서 지역업체인 슬롯은 `1.5`
- 지역업체가 아니면 `0`
- 최종 점수는 지분 반영으로 계산
- 최종 max는 `0.3`

이 `1.5`는 임의값이 아니라 아래 식을 공통 흐름에 맞게 변환한 값이다.

- 원식:
  `0.30 × (지역업체 참여비율 / 20%)`
- 변환:
  `0.30 / 20 = 0.015`
- 지분이 `%` 단위이므로 슬롯 기준값:
  `0.015 × 100 = 1.5`

즉 예:

- 지역업체 지분 `10%` → `1.5 × 0.1 = 0.15`
- 지역업체 지분 `20%` → `1.5 × 0.2 = 0.30`
- `20% 이상`이면 최종 `0.3`으로 clamp

### 3-3. 같은 결과 반복 저장 방지

`AgreementBoardWindow.jsx`에 `equalGroupSummaries()`를 추가했다.

현재는:

- `computeGroupSummaries()` 결과가 이전과 같으면
- `setGroupSummaries(results)`를 다시 하지 않음

이 가드는 간이종심제 전용 우회가 아니라, 공통적으로 안전한 방향이다.

## 4. 현재 간이종심제 상태 정리

현재 간이종심제는 아래 방향으로 정리된 상태다.

- 공통 계산 흐름 사용
- 공통 `group-summary` 사용
- 간이종심제 전용 별도 흐름 제거
- 기준값만 간이종심제 버전으로 적용

남아 있는 간이종심제 기준 차이:

- 실적 만점 `11`
- 만점 기준 실적 `기초금액 × 3`
- 지역경제 기여도 max `0.3`
- 기타점수 `17`
- 최종 만점 `40`

즉 현재는 `간이종심제만 별도 기능으로 완전히 분리해야 하는 상태`는 아니고,
다른 발주처처럼 공통 구조 위에 기준만 다르게 얹는 방향으로 유지 가능하다.

## 5. LH 50억 미만 / 50억~100억 관련 같이 수정한 내용

이번 세션에서 간이종심제 외에 LH 기존 구간도 같이 보정했다.

### 5-1. 실적점수 반올림 기준 수정

사용자 확인 결과, LH 엑셀은 아래 방식이었다.

- `실적비율`을 먼저 `ROUND(..., 2)`
- 그 다음 `× 13`

프로그램은 반대로:

- `실적비율 × 13`
- 마지막 결과 반올림

이라 값 차이가 발생했다.

수정 결과:

- `LH 50억 미만`
- `LH 50억~100억`

둘 다 현재는 엑셀과 동일하게:

- `ROUND(실적비율, 2) × 13`

방식으로 계산한다.

관련 파일:

- `src/shared/agreements/calculations/performanceScore.js`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

### 5-2. LH 50억~100억 실적 기준금액 수정

사용자 기준:

- `LH 50억~100억` 실적 기준은 무조건 `기초금액`
- `추정금액` 사용 아님

수정:

- `group-summary`
- 후보 재평가

둘 다 `LH 50억~100억`에서는 `perfBase = baseAmount`로 고정

### 5-3. 3년 실적 / 5년 실적 규칙 정리

사용자 기준:

- `행안부 50억~100억`만 `3년 실적`
- 그 외 전부 `5년 실적`

수정:

- 공통 실적 추출 로직 기본값은 `5년 실적`
- `행안부 50억~100억`만 예외적으로 `3년 실적`

관련 파일:

- `src/shared/agreements/calculations/performanceValue.js`

### 5-4. LH 50억~100억 엑셀 내보내기 보정

수정 내용:

- 신인도 점수 컬럼 추가
  - 협정1 기준 `AF5`
  - 협정2 기준 `AF7`
- 업체명 아래 `가능지분` 표시 포함

관련 파일:

- `main.js`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

## 6. 후보 재조회 최신정보 반영 문제

증상:

- 업체를 협정보드에 올린 뒤 최신 정보로 갱신
- 협정테이블에서 제거하면 후보로 돌아감
- 후보에서도 최종 제거 후 다시 조회해도 예전 정보가 섞여 보임

원인:

- 같은 업체를 다시 조회했을 때 기존 candidate와 merge하면서
  이전 `snapshot`/예전 값이 남는 구조

수정:

- 같은 `id` 후보를 다시 추가하면
  기존 후보 데이터를 넓게 merge하지 않고
  새 조회 결과를 기준으로 덮어씀
- 단, 후보 보관함 표시 플래그 `_agreementCandidateListed`만 유지

관련 파일:

- `src/view/features/agreements/context/AgreementBoardContext.jsx`

## 7. 순공사원가가점 수동 입력 보정

수정 목적:

- LH 일반 구간에서 자동 계산값이 있어도
  사용자가 수동으로 순공사원가가점을 입력 가능해야 함

현재 구조:

- 비워두면 자동 계산값 사용
- 사용자가 입력하면 `netCostBonusOverride` 우선 사용
- 총점 계산에도 입력값 반영

주의:

- 간이종심제에는 순공사원가가점 없음
- 간이종심제 쪽으로는 확장하지 않음

관련 파일:

- `src/view/features/agreements/context/AgreementBoardContext.jsx`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

  품질 셀 배경을 노란색으로 표시

기준:

- `업체 품질점수 > lhQualityDefault`

적용 파일:

- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

참고:

- 간이종심제 품질총점은 별도 반올림 없이
  `품질점수 × 지분` 합으로 계산
- 화면에는 소수점 2자리로만 표시
- 품질 환산점수 구간 판정은 반올림 전 원값 기준

## 9. 현재 확인된 최종 상태

간이종심제 기준 현재 확인된 상태:

- `LH 100억~300억` 선택 시 더 이상 즉시 멈추지 않음
- 지역경제 기여도 점수 다시 계산됨
- `20% 이상이면 0.3점` 상한도 코드상 맞음
- 공통 흐름을 타고 기준값만 간이종심제 기준으로 적용 중

즉 현재 결론:

- 간이종심제를 별도 기능으로 통째로 분리할 필요성은 낮아짐
- 공통 계산 구조를 유지한 채 기준만 다르게 두는 현재 방향이 맞음
