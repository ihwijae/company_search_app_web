## 1. 국가철도공단 50억미만 소방 실적점수 버그

### 증상

- `국가철도공단 / 50억미만 / 소방` 협정보드에서 실적점수가 만점이 아닌데도 전부 `15점`으로 보임
- 사용자 예시:
  - A업체 5년실적 `1,555,486,000`, 지분 `70%`
  - B업체 5년실적 `3,636,972,000`, 지분 `30%`
  - 기초금액 `3,229,592,000`
- 위 값이면 정상 점수는 `3.30점` 또는 `5.10점`이어야 하는데, 협정보드에서는 `15점` 표시

### 원인

디버그 결과:

- 가중실적합은 정상
  - `2,179,931,800`
- 기초금액/추정가격도 정상
  - `3,229,592,000`
- 그런데 `summary.performanceRatio = 0.67`

즉 앱이 아래처럼 계산하고 있었음.

- 잘못된 계산: `실적합계 / 기초금액`

하지만 국가철도공단 50억미만 소방 기준은 아래여야 함.

- 추정가격 `30억 미만`:
  - `실적합계 / (기초금액 × 2)`
- 추정가격 `30억 이상`:
  - `실적합계 / (기초금액 × 3)`

정리하면 공통 실적점수 계산 경로가 이 예외 규칙을 반영하지 못한 것이 원인이다.

### 수정

수정 파일:

- `src/shared/agreements/calculations/performanceScore.js`

처리 방식:

- 공통 실적점수 계산 함수 안에서 아래 조건에만 예외 분기 추가
  - `agencyId === 'krail'`
  - `fileType === 'sobang'`
  - `50억 미만 구간`
- 이 경우 직접 계산
  - `30억 미만`: `ROUND(실적합계 / (기초금액 × 2), 2) × 15`
  - `30억 이상`: `ROUND(실적합계 / (기초금액 × 3), 2) × 15`
- 결과는 최종 `15점` cap 유지

### 참고

- 이번 수정은 `국가철도공단 50억미만 소방`만 별도 분기 처리
- 다른 발주처 전체를 한 번에 고친 것은 아님
- 추후 `공종/금액에 따라 분모 배수`가 달라지는 다른 발주처도 같은 방식 점검 필요

## 5. 카카오톡 전송 페이지 수정

### 5-1. 메시지 수정 상태(`수정됨`) 오표시

#### 증상

- 메시지 템플릿을 바꾸지 않았는데도 `수정됨` 표시가 남아 있음
- 사용자가 `직접 입력`으로 다시 저장해도 `수정됨`이 계속 표시됨
- 하지만 실제 문자 생성 결과는 정상처럼 보임

#### 원인

- `messageOverrides[entry.id]` 키가 존재하는지만 보고 `수정됨` 표시를 하던 구조
- 기본 메시지와 내용이 같아도 override 키가 남아 있으면 계속 `수정됨`

#### 수정

수정 파일:

- `src/view/features/kakao/pages/KakaoSendPage.jsx`

수정 내용:

- 기본 메시지와 동일한 override는 저장 시 삭제
- 오래된 no-op override 자동 정리
- 버튼 표시와 묶음 메시지도 실제 차이 기준으로 판정

### 5-2. 공종 선택 위치/동작 수정

#### 요구사항

- 공종을 먼저 고르고 `문자 분리`를 눌러야 하는데, 공종 선택 UI가 `업체별 담당자 목록` 쪽에 있어 순서가 어색했음
- `공종 자동` 옵션 제거 필요
- 공종 미선택 상태에서 `문자 분리` 누르면 우리 UI 팝업으로 안내 필요

#### 수정

수정 파일:

- `src/view/features/kakao/pages/KakaoSendPage.jsx`

수정 내용:

- 공종 선택 메뉴를 `협정문자 입력` 영역, `문자 분리` 버튼 근처로 이동
- `공종 자동` 삭제
- 공종 선택 없이 `문자 분리` 또는 `담당자 자동매칭` 실행 시 전역 피드백 팝업(`confirm`)으로 안내

## 6. 임시 업체 등록 기능

### 6-1. 개요

협정보드에서 DB에 없는 업체도 임시로 등록해 검색/협정에 활용할 수 있도록 기능 추가.

요구사항:

- 업체검색 모달 좌하단 `업체추가` 버튼
- 별도 창에서 임시 업체 관리
- DB와 동일한 필드 사용
- 사용자가 삭제하기 전까지 유지
- 팀원 공유를 위한 가져오기/내보내기

### 6-2. 저장 구조

내부 저장:

- `SQLite`
- 파일명: `temp-companies.sqlite`

외부 공유:

- `JSON`

충돌 규칙:

- `업체명 + 사업자번호`가 같으면 가져오기 시 덮어쓰기

### 6-3. 데이터 필드

임시 업체 필드는 아래와 동일하게 구성.

- 업체명
- 대표자
- 사업자번호
- 지역
- 시평
- 3년실적
- 5년실적
- 부채비율
- 유동비율
- 영업기간
- 신용평가
- 여성기업
- 중소기업
- 일자리창출
- 품질평가
- 비고

### 6-5. 검색/표시 동작

- 임시 업체도 기존 DB 검색 결과와 함께 보여줌
- 중복 업체가 있어도 둘 다 표시
- 업체검색 모달, 협정보드 쪽에 `임시` 배지 표시

### 6-6. 지역 판정 동작

임시 업체도 지역업체 판정 가능함.

이유:

- 임시 업체 검색 결과 생성 시
  - `지역`
  - `대표지역`
  둘 다 임시 업체 `region` 값으로 채움
- 협정보드 지역 판정은 시트 위치가 아니라 이 문자열을 기준으로 비교

관련 코드:

- `src/main/features/temp-companies/service.js`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

현재 판정 방식:

- 공백 제거 후 비교
- 완전일치면 통과
- `startsWith` 비교도 허용

예:

- 임시 업체 지역 `경기도 안산시`
- 의무지역 `경기` 또는 `경기도`
- 현재 코드상 지역업체로 인식될 수 있음

### 6-7. 별도 창 방식 변경

처음에는 Electron 메인 프로세스의 `BrowserWindow`로 구현했음.

하지만 사용자 요구사항은 협정보드처럼 `window.open(...)` 기반 별도 창이었음.

최종 변경:

- `업체추가` 버튼 클릭 시 렌더러에서 `window.open('#/temp-companies')` 방식으로 창 오픈
- 즉 협정보드와 같은 계열의 팝업 창 방식으로 변경

수정 파일:

- `src/components/CompanySearchModal.jsx`

### 6-8. 새 창 API 연결 문제

#### 증상

- 임시 업체 창에서 저장 시
  - `tempCompanies API method save is not available`

#### 원인

- `window.open`으로 연 새 창에는 `window.electronAPI.tempCompanies`가 직접 없었음

#### 수정

수정 파일:

- `src/shared/tempCompaniesClient.js`

수정 내용:

- 현재 창에 API가 없으면 `window.opener.electronAPI.tempCompanies`를 fallback으로 사용

### 6-9. 입력 포맷

#### 콤마 포맷 적용

적용 필드:

- 시평
- 3년실적
- 5년실적
- 품질평가

예:

- `1000000` 입력 -> `1,000,000`

#### 부채비율 / 유동비율

DB 구조와 맞추기 위해 마지막 두 자리를 소수로 처리

예:

- `5605` -> `56.05`
- `12535` -> `125.35`

#### 영업기간

영업기간은 숫자 포맷 대상에서 제외

형식 예:

- `1997.10.01`

### 6-10. UI 조정

수정 내용:

- 상단 검은 줄(`title-drag`) 제거
- 비고 칸을 하단 전체폭이 아니라 품질평가 옆 칸에 배치

현재 임시 업체 수정 방식:

- 왼쪽 목록에서 기존 임시 업체 선택
- 값 수정
- 다시 `저장`

즉 별도 `수정` 버튼은 없고, 선택된 항목 저장이 곧 수정이다.
