# 개발 서버 세팅 정리

## 1. 목적
- 개발 PC(WSL)에서 웹 기능을 빠르게 테스트하면서 로그인/API까지 정상 동작하도록 구성한다.

## 2. 현재 개발 실행 방식
- 명령어: `npm run start:dev`
- 동작:
  1. `npm run serve:lan` 실행 (포트 `4173`)
  2. `vite --host 0.0.0.0` 실행 (포트 `5173`)

즉, 개발 시 두 서버가 동시에 뜬다.

## 3. 포트 역할
- `5173`: Vite 개발 프론트 서버 (HMR/빠른 반영)
- `4173`: Node 서버 (`server.js`) + API(`/api/*`) 처리

## 4. 로그인 404 원인과 해결

### 4-1. 원인
- 브라우저를 `5173`으로 접속했는데 `/api/auth` 요청이 `4173`으로 전달되지 않으면 `404`가 발생한다.

### 4-2. 해결
- `vite.config.js`에 `/api` 프록시를 추가하여 `4173`으로 전달되게 설정.

```js
server: {
  port: 5173,
  strictPort: true,
  proxy: {
    '/api': {
      target: 'http://127.0.0.1:4173',
      changeOrigin: true,
    },
  },
},
```

## 5. package.json 변경 사항
- `start:dev`를 아래처럼 변경:

```json
"start:dev": "concurrently \"npm run serve:lan\" \"vite --host 0.0.0.0\""
```

## 6. 접속 방법 (Windows 브라우저)
1. WSL에서 IP 확인:
   - `ip -4 addr show eth0`
2. 예시 IP: `172.22.197.199`
3. 브라우저 접속:
   - `http://172.22.197.199:5173`

## 7. 운영 서버와 차이
- 운영 서버는 `serve:prod:lan`으로 `4173` 하나에서 프론트+API를 함께 제공한다.
- 개발 서버는 `5173`(프론트) + `4173`(API) 분리 구조다.

## 8. 빠른 체크 명령어

### 8-1. 포트 확인
```bash
ss -lntp | grep -E "5173|4173"
```

### 8-2. API 확인
```bash
curl -I http://127.0.0.1:4173/api/auth?action=session
```

### 8-3. 프론트 확인
```bash
curl -I http://127.0.0.1:5173
```

## 9. 요약
- 개발은 `npm run start:dev` 한 번으로 충분.
- 브라우저는 `5173`으로 접속.
- `/api`는 Vite 프록시를 통해 자동으로 `4173`에 연결됨.
