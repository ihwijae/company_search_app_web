# 리팩토링 기능 분리 정리 (2026-03-06)

## 목적

`AgreementBoardWindow.jsx`에 몰려 있던 계산/조립/배치 로직을 기능별 파일로 분리해서:

- 메인 view 파일 복잡도 감소
- 계산 기준 단일화
- 후보/협정테이블/엑셀/지역사찾기 간 기준 통일
- 다음 세션에서 기능 수정 시 영향 범위 축소

를 목표로 진행했다.

---

## 현재 구조 기준

- `src/view/features/agreements/...`
  - 화면, 이벤트 연결, confirm/alert, React state

- `src/shared/agreements/calculations/...`
  - 공용 계산 로직
  - 값 추출/표시용 selector/helper
  - 점수 계산

- `src/shared/agreements/runner/...`
  - 계산 실행 흐름

- `src/shared/agreements/placement/...`
  - 협정테이블 배치/교체/제거 관련 상태 변경 로직

- `src/main/features/agreements/...`
  - Electron main process
  - 엑셀 실제 쓰기 등 Node/Electron 책임

### 10. 실적점수 계산 본체

- `src/shared/agreements/calculations/performanceScore.js`

역할:

- “점수 계산 함수 본체”를 먼저 분리한 단계였음

---

### 11. 후보 점수 재계산 운영 흐름

- `src/shared/agreements/runner/candidateScoreRunner.js`

역할:
- 어떤 후보를 재계산할지 판단
- cache key 생성/캐시 상태 처리
- formulasEvaluate payload 조립
- 경영/실적 결과를 candidate에 반영
- fallback 점수 반영

포함 함수:
- `runAgreementCandidateScoreEvaluation()`

메모:
- 메인 파일에는 effect, 취소 처리, version 갱신만 남김

---

### 12. 협정테이블 배치/교체/제거 상태 변경

- `src/shared/agreements/placement/boardPlacement.js`

역할:
- 협정테이블 상태 변경 본체
- 이동/교체/드롭 관련 matrix 변경
- 일부 이동 분기 판단/문구 조합

현재 포함 함수:
- `placeEntryInAssignments()`
- `resetSlotInMatrix()`
- `placeEntryOnBoard()`
- `swapOrMoveBoardEntries()`
- `applyDropToAssignments()`
- `createPendingMoveSource()`
- `shouldResetPendingMove()`
- `buildBoardMoveConfirmPayload()`
- `buildCandidatePlacementNotice()`

주의:
- confirm 호출, alert 호출, React state setter 등 UI 연결층은 아직 `AgreementBoardWindow.jsx`에 남아 있음

## 지역사찾기 기능 반영

수정 파일:
- `src/view/features/agreements/components/CandidatesModal.jsx`

반영 내용:
- 지역사찾기에서 자체적으로 하던 `경영점수 총점 계산` 제거
- 협정보드와 동일하게 `managementScore.js` 사용

현재 상태:
- 경영점수 총점 계산 기준은 협정보드와 동일
- 부채/유동/신용 breakdown 표시 UI는 그대로 유지
- 불필요한 옛 총점 계산 흔적 일부 정리 완료

중요:
- 지역사찾기 view 구조는 유지
- 계산 기준만 공용화

## 현재 의미 있는 상태

### 핵심 계산 로직

대부분 분리됨:

- 경영점수 계산
- 실적 값 선택
- 실적점수 계산 본체
- 시평액 값 추출
- 가능지분 계산
- 그룹 summary 계산

### 이동 관련 로직

핵심 로직은 대부분 분리됨:

- 배치
- 교체
- 제거
- 드래그 드롭 상태 변경

하지만 아래는 메인 view에 남아 있음:

- confirm 호출
- alert 호출
- React state 연결
- 후보복귀 호출 일부

즉:
- 핵심 로직은 밖에 있음
- UI 연결층만 view에 남아 있는 구조

## 인계용 한 줄 요약

협정보드의 핵심 계산 로직과 배치 상태 변경 로직은 상당 부분 `shared`로 분리된 상태다.  
다음 세션에서는 `복사 기능` 또는 `보드 검색`을 우선 분리하는 것이 가장 안전하고 효율적이다.
