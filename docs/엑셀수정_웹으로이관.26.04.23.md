# 엑셀수정 웹 이관 정리 (2026-04-23)

## 1. 문서 목적
- 2026-04-23 기준으로 엑셀수정(웹) 이관 진행 사항을 정리한다.
- 기존 Python GUI(`clone/excel_modifi`) 기준 업무 흐름과 비교해 현재 웹 구현 상태를 명확히 남긴다.

## 2. 이관 범위 기준
- 유지 대상:
  - 경영상태 수정/저장
  - 신용평가 수정/저장
  - 사업자번호 조회
  - 파일 보관(스캔본 저장)
  - 연말 색상 업데이트
  - 신용평가 유효기간 갱신
- 제외 대상:
  - OCR 자동 인식/ROI 지정
  - SMPP(여성/중소기업 외부 연동)

## 3. 아키텍처/경로

### 3.1 백엔드
- FastAPI: `backend/python/app/main.py`
- 기본 실행 포트: `127.0.0.1:8787`
- 주요 엔드포인트:
  - `POST /excel-edit/upload`
  - `POST /excel-edit/company-lookup`
  - `POST /excel-edit/save`
  - `POST /excel-edit/update-year-end-color`
  - `POST /excel-edit/update-credit-expiry`
  - `POST /excel-edit/render-pdf-page`
  - `POST /excel-edit/render-image`
  - `POST /excel-edit/export-pdf-pages`
  - `POST /excel-edit/remove-pdf-pages`

### 3.2 프론트
- 페이지: `src/view/features/excel-web/pages/ExcelWebEditPage.jsx`
- 클라이언트: `src/shared/excelEditBackendClient.js`
- 스타일: `src/styles.css`

### 3.3 데이터/파일 경로 정책
- DB는 업로드된 웹 기준 DB(`uploads/master-files`)를 사용.
- 스캔본 보관 루트는 `EXCEL_EDIT_ARCHIVE_ROOT` 환경변수 우선.
- 지역 폴더명은 별칭 정규화 적용:
  - 예: `경기도 -> 경기`, `대구광역시 -> 대구`
- 파일명 충돌 시 기존 파일 삭제 후 덮어쓰기 저장.

## 4. 완료된 기능

### 4.1 조회/저장 기본 흐름
- 사업자번호로 업체 조회 가능.
- 저장 시 경영상태/신용평가 각각 엑셀 반영 가능.
- 저장 후 화면 상태 초기화(업로드 목록, 폼, 미리보기) 처리.
- 저장 후 업로드 DB 재색인(refresh) 연동으로 검색 데이터 즉시 반영.
- 동시 수정 안정화를 위해 파일 락 + 버전 검증(낙관적 동시성) 반영.
  - 조회 응답에 `version` 포함
  - 저장 요청에 `expectedVersion` 포함
  - 불일치 시 `409` 충돌로 저장 차단(재조회 유도)

### 4.2 미조회 분기(GUI 방식)
- `불러오기` 시 업체 미조회면 분기 창 노출.
- 경영상태:
  - `신규업체등록`
  - `파일만저장`
- 신용평가:
  - `파일만저장` 중심 흐름
- `신규업체등록` 선택 시 업체명/지역 입력 -> 우측 입력폼 반영 후 저장.
- `파일만저장` 선택 시 엑셀 수정 없이 파일만 규칙대로 보관.

### 4.3 입력 편의 기능
- 사업자번호 자동 하이픈 포맷: `1234567890 -> 123-45-67890`
- 시평액/3년실적/5년실적 자동 천단위 콤마.
- 부채비율/유동비율 자동 `%` 포맷(`51212 -> 512.12%`).
- 시작일/종료일 자동 날짜 포맷(`YYYYMMDD -> YYYY.MM.DD`).
- 영업기간도 동일 날짜 포맷 적용.
- 금액 필드 저장 시 천원 단위 입력값을 실제 엑셀값으로 `*1000` 반영.

### 4.4 미리보기/뷰어
- PDF/이미지 Python 렌더 미리보기.
- PDF 페이지 이동/확대/축소 지원.
- 뷰어 90도 회전(좌/우) 및 회전 초기화 지원.
- PDF가 2페이지 이상일 경우 확인 모달 강제 노출(사용자 확인 후 닫힘).

### 4.5 PDF 페이지 내보내기(중요)
- 페이지 범위 입력으로 선택 페이지만 별도 PDF 다운로드.
- 내보내기 직후 현재 뷰어 원본 PDF에서 해당 페이지 자동 삭제.
- 삭제 후 자동 재로드.
- 남은 페이지가 0이면 목록에서 자동 제거.

### 4.6 운영성 갱신 기능
- `연말 색상 업데이트` 실행 가능.
- `신용평가 유효기간 갱신` 실행 가능.
- 둘 다 엑셀수정 화면 `갱신기능` UI에서 호출.

### 4.7 알림/UI 정리
- 엑셀수정 화면 알림을 공통 UI 알림(Feedback toast)로 통일.
- 업체 미조회/업체정보 입력 창은 사용자 작업성 위해 플로팅 창(UI 창) 방식으로 운영.
- 폰트 전반 확대 적용.
- 자료종류 색상 표시:
  - 전기경영상태: 노랑
  - 통신경영상태: 하늘
  - 소방경영상태: 빨강

## 5. 스캔본(파일 탐색) 관련 반영
- 스캔본 메뉴 제공(폴더 탐색/미리보기/다운로드/전체 ZIP).
- 검색 기능 개선:
  - 실시간 검색
  - 전체 폴더 대상 검색
  - 검색 시 해당 폴더로 자동 이동
- 필터 제공:
  - 전기/통신/소방/신용평가
- `:Zone.Identifier` 메타 파일 제외 처리.

## 6. 운영/배포 관련 메모
- 운영에서 Python 백엔드는 별도 프로세스로 상시 실행 필요.
- 터널/프록시 환경에서는 `/api/excel-edit/*`가 Python 백엔드로 정상 프록시되도록 설정 필요.
- `npm run start:dev`는 로컬 개발용이며, 운영은 PM2/systemd 기준 실행 권장.

## 7. 남은 과제(후속)
- 2차 이관 후보:
  - 업체명 기반 조회
  - 변경 이력(audit log) 조회/관리
- 실 운영 데이터로 최종 시나리오 검증:
  - 경영상태: 기존업체 수정 / 신규업체등록 / 파일만저장
  - 신용평가: 기존업체 수정 / 파일만저장
  - PDF 분할 내보내기 + 원본 페이지 제거 흐름
