# 엑셀수정 웹 이관 정리 (2026-04-24)

## 1. 오늘 반영된 수정사항

### 1.1 부채비율/유동비율 백스페이스 입력 버그 수정
- 증상:
  - `부채비율`, `유동비율` 입력칸에서 숫자가 잘 지워지지 않음(백스페이스 체감 불가).
- 원인:
  - `%` 자동 포맷이 매 입력마다 재적용되어 `%` 삭제가 즉시 복원됨.
- 조치:
  - `deleteContentBackward` 입력 타입을 감지해 `%` 삭제 상황을 마지막 숫자 1자리 삭제로 처리.
- 수정 파일:
  - `src/view/features/excel-web/pages/ExcelWebEditPage.jsx`

### 1.2 뷰어 렌더 지연 1차 완화(중복 요청/경합 제거)
- 목적:
  - 새 파일 첫 로드/파일 전환 시 불필요한 렌더 요청과 경합 감소.
- 조치:
  - PDF 첫 로드 직후 발생하던 동일 페이지 중복 렌더 호출 1회 스킵.
  - `renderPdfPage`, `renderImage` 요청에 `AbortController` 적용.
  - 파일 전환/해제/언마운트 시 진행 중 렌더 요청 취소.
  - 취소 에러(`AbortError`)는 사용자 오류 알림으로 노출하지 않음.
- 수정 파일:
  - `src/view/features/excel-web/pages/ExcelWebEditPage.jsx`
  - `src/shared/excelEditBackendClient.js`

## 2. 현재 상태 평가
- 이번 1차 수정은 "불필요한 추가 지연"을 줄이는 안정화 성격.
- 새 파일 첫 로드 지연의 본체는 여전히 다음 비용이 큼:
  - 렌더 요청마다 파일 본문 전송
  - Python 렌더링(PDF pixmap/이미지 변환) 자체 비용

## 3. 다음 세션 권장 작업(우선순위)

### 3.1 우선 검토
- 미리보기 기본 DPI 하향(예: 160 -> 120 또는 140)
  - 장점: 즉시 체감 개선
  - 단점: 미리보기 선명도 일부 저하

### 3.2 구조 개선(2차)
- `fileId` 기반 렌더 구조 도입 검토
  - 최초 1회 임시 업로드 후 페이지 렌더는 `fileId + page + dpi`만 전송
  - 페이지 이동이 잦은 PDF에서 효과 큼
  - 임시 저장 TTL/정리 정책 필요

## 4. 검증/제약 메모
- 로컬 빌드 검증 시도:
  - `npm run build` 실패 (환경 의존 모듈 누락)

- 따라서 이번 세션은 정적 빌드 성공 확인까지는 미완료.
