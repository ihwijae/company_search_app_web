# 엑셀수정 웹 전환 정리 (2026-04-27)

## 1. 오늘 반영된 수정사항

### 1.1 경영상태 업체 삭제 기능 추가
- 범위:
  - `전기경영상태`
  - `통신경영상태`
  - `소방경영상태`
- 제외:
  - `신용평가` 삭제는 이번 작업 대상에서 제외
- 삭제 방식:
  - 엑셀 `행 삭제`가 아니라 `업체 블록 값 비우기` 방식으로 구현
  - 사업자번호로 업체 위치를 찾은 뒤, 해당 업체가 속한 `회사명 ~ 비고` 구간만 비움
  - 스타일 구조를 흔들지 않고 빈 슬롯으로 되돌리는 목적
- 안정성:
  - 기존 저장과 동일하게 파일 락 적용
  - `expectedVersion` 기반 버전 검증 유지
  - 조회 후 다른 사용자가 먼저 수정한 경우 삭제 차단
- 백엔드 추가:
  - `POST /excel-edit/delete-company`
- 수정 파일:
  - `backend/python/app/main.py`
  - `src/shared/excelEditBackendClient.js`

## 2. 스캔본 기능 수정

### 2.1 검색 결과가 첫 번째 파일로만 이동하던 흐름 수정
- 기존 문제:
  - 검색어와 부분 일치하는 파일이 여러 개여도
  - 첫 번째 결과 1개만 기준으로 해당 폴더로 바로 이동
  - 사용자 입장에서는 다른 검색 결과가 없는 것처럼 보임
- 실제 사례:
  - `(유)우전_전기경영상태.pdf`
  - `제우전력(주)_신용평가.pdf`
  - `우전` 검색 시 두 파일이 모두 매칭될 수 있으나, 기존 UI는 첫 결과만 사용
- 원인:
  - 검색 로직 자체는 정상적으로 다건 결과를 만들고 있었으나
  - 화면이 `result.results[0]`만 사용하고 자동 이동하고 있었음
- 변경:
  - 검색 시 `검색 결과` 목록을 별도로 표시
  - 더 이상 첫 결과로 자동 이동하지 않음
  - 사용자가 원하는 파일을 클릭하면 그때 해당 폴더로 이동하고 파일 선택
  - 검색 결과 목록에는 파일명과 폴더 경로를 함께 표시
- 효과:
  - `(유)` 같은 접두어 때문에 검색이 안 되는 것처럼 보이던 오해 해소
  - 부분 일치 파일이 여러 개인 경우 직접 선택 가능
- 수정 파일:
  - `src/view/features/scan-archive/pages/ScanArchivePage.jsx`
  - `src/styles.css`

### 2.2 스캔본 파일명 폰트 크기 확대
- 문제:
  - 파일명 텍스트가 작아 가독성이 떨어짐
- 변경:
  - 파일명 폰트 크기와 줄간격 확대

## 3. 개발/배포 환경 이슈 메모

### 3.1 개발 PC 포맷 이후 Node/Python 환경 재설정 필요 확인
- `npm run start:dev` 실행 시 `concurrently: not found` 발생
- 원인:
  - `node_modules` 부재
- 조치:
  - `npm install` 재실행 필요 확인

### 3.2 WSL 기본 유틸 누락 이슈
- `npm install` 중 `phantomjs-prebuilt` 압축 해제 실패
- 원인:
  - `bzip2` 미설치
- 조치:
  - `apt install bzip2` 필요 확인

### 3.3 Python 가상환경 구성 이슈
- `npm run start:dev`에서 Python 백엔드가 `./.venv/bin/uvicorn: not found`로 실패
- 원인:
  - `backend/python/.venv` 미생성
- 추가 원인:
  - `python3 -m venv .venv` 시 `python3-venv` 미설치 상태
- 조치:
  - `python3.x-venv` 설치 후 `.venv` 재생성 필요 확인
  - 이후 `fastapi`, `uvicorn`, `openpyxl`, `pymupdf`, `pillow`, `python-multipart` 설치 필요 확인

### 3.4 자동배포 실패 원인 확인 및 재발 방지 수정
- 증상:
  - GitHub Actions 배포에서 `git pull --ff-only origin main` 실패
- 원인:
  - Python 실행 중 생성된 `backend/python/app/__pycache__/*.pyc` 파일이 저장소 추적 대상이었음
  - 서버 작업 디렉토리에 남아 있던 캐시 파일이 pull 과정에서 충돌
- 반영:
  - `.gitignore`에 Python 캐시 제외 규칙 추가
    - `__pycache__/`
    - `*.py[cod]`
  - 배포 스크립트 시작 시 `__pycache__`, `.pyc`, `.pyo` 삭제하도록 보강
- 수정 파일:
  - `.gitignore`
  - `scripts/deploy-prod.sh`

## 4. 현재 상태 정리
- 엑셀수정(경영상태)은 이제 `조회 / 수정 / 신규등록 / 파일만저장 / 삭제` 흐름까지 웹에서 처리 가능
- 삭제는 선택한 경영상태 DB 하나만 대상으로 하며, 신용평가 삭제는 아직 제외
- 스캔본 검색은 이제 전역 검색 결과를 사용자에게 명시적으로 보여주는 구조로 보완됨
- 개발 PC 재설치 이후에는 Node/Python/WSL 기본 패키지 복구가 함께 필요함

## 5. 검증 메모
- 업체 삭제 기능은 실제 UI에서 삭제 동작 확인 완료
- 스캔본 검색 수정은 UI에서 다건 검색 결과가 함께 표시되는지 추가 확인 필요
- Python 배포 캐시 문제는 서버에서 기존 추적 캐시 파일 삭제 커밋 반영 여부까지 확인 필요
