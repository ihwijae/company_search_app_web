# 엑셀수정 기능 웹 전환 정리

## 1. 문서 목적
- 기존 파이썬 GUI(`clone/excel_modifi`)에서 운영 중인 엑셀 수정 업무를 웹으로 이관하기 위한 합의사항을 정리한다.
- 이번 문서는 대화에서 확정된 범위와 제외 범위를 기준으로, 이후 구현 시 기준 문서로 사용한다.

## 2. 배경
- 현재 운영 방식은 담당자 1인이 로컬 환경에서 DB 엑셀(전기/통신/소방)과 증빙 파일(PDF/이미지)을 직접 관리하는 구조다.
- 팀원별 추가/수정 업무가 발생해도 중앙 협업이 어렵고, 파일 공유/이력 관리가 비효율적이다.
- 웹 이관을 통해 팀 단위 협업, 공용 저장소 기반 파일 관리, 변경 이력 관리를 목표로 한다.

## 3. 이번 대화에서 확정된 결정사항

### 3.1 이관 방향
- 기존 기능을 웹으로 이관한다.
- UI는 웹 기준으로 재설계한다. (PySide UI 재사용 아님)
- 비즈니스 규칙은 가능한 한 기존 동작과 동일하게 유지한다.

### 3.2 제외 기능 (이번 이관 범위에서 제외)
- `SMPP 기반 여성기업/중소기업 조회` 기능 제외
  - 즉, `smpp_*` 계열 기능은 웹 이관 범위에서 제거
- `OCR 자동 인식` 기능 제외
  - 실제 운영에서 OCR 편차가 커 수동 입력 중심으로 사용 중인 점을 반영

### 3.3 유지 기능 (필수)
- 경영상태 데이터 수동 입력/수정
- 신용평가 데이터 수동 입력/수정
- 파일 업로드/보관(증빙 파일 관리)
- `연말 색상 업데이트` 기능 유지
- `신용평가 유효기간 갱신` 기능 유지

## 4. 기존 파이썬 프로젝트 기준 핵심 업무 흐름(유지 대상)
1. 자료(파일) 선택
2. 사업자번호 기준 업체 조회
3. 기존 업체 수정 또는 신규 업체 추가
4. 저장 시 데이터 반영 + 파일 보관
5. 운영성 갱신 기능(연말 색상 업데이트, 신용평가 유효기간 갱신)

## 5. 웹 전환 시 설계 원칙
- 로컬 파일 중심이 아닌 서버 저장소 중심으로 전환한다.
- DB를 단일 기준 데이터 소스로 두고, 엑셀 파일은 운영 데이터 소스가 아닌 보조 산출물/이관 자료로 취급한다.
- 누가 언제 무엇을 변경했는지 이력(audit log)을 남긴다.
- 기능을 단순화해 초기 릴리스는 수동 입력/검수 중심으로 구성한다.

## 6. MVP(1차) 범위 제안
- 업체 검색/조회(사업자번호, 업체명)
- 경영상태 입력/수정 화면
- 신용평가 입력/수정 화면(유효기간 포함)
- 첨부파일 업로드/목록/다운로드
- 연말 색상 업데이트 실행 버튼
- 신용평가 유효기간 갱신 실행 버튼
- 변경 이력 기본 조회

## 7. 비범위(Out of Scope)
- SMPP 외부 사이트 연동/크롤링
- OCR ROI 지정/자동 추출
- 데스크톱 전용 UI/동작 호환

## 8. 구현 시 주의사항
- 기존 파이썬 로직에서 사용하던 필드 의미(사업자번호 기준 키, 신용평가 문자열 포맷, 색상 갱신 기준)를 정확히 매핑해야 한다.
- 기존 운영 데이터(전기/통신/소방) 마이그레이션 전략을 먼저 확정해야 한다.
- 파일 저장 경로 정책(업체명/지역 기반 네이밍)을 서버 저장소 정책으로 재정의해야 한다.

## 9. 결론
- 웹 이관은 진행한다.
- OCR과 SMPP는 제외하고, 수동 입력 + 파일 관리 + 갱신 기능 2종(연말 색상 업데이트, 신용평가 유효기간 갱신)을 중심으로 구현한다.
- 즉, 이번 전환의 핵심은 자동화 확장보다 협업 가능한 운영 시스템으로의 전환이다.
