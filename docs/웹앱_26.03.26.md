# 웹앱 전환 인계 문서 (2026-03-26)

## 1) 목표
- 기존 GUI(Electron) 기능을 웹(Vercel) 기준으로 단계적 이관
- 우선순위: 업체조회/협정보드 핵심 기능 안정화 -> 나머지 기능 이관

## 2) 오늘까지 완료된 핵심 작업

### A. 업체조회(웹) 공용 데이터화
- 엑셀 업로드를 서버 저장소(Vercel Blob) 기반으로 변경
- 공종(전기/통신/소방) 업로드/상태/검색/지역 API 구성
- 통신/소방 파싱 중 `comments` 관련 오류 대응
- 지역 목록 API(`/api/regions`) 응답 파싱 이슈 수정

### B. 협정보드 저장/불러오기 웹 전환
- 협정 저장/목록/불러오기/삭제를 Blob 기반 API로 구현
- 새 탭 기반 구조로 변경(팝업 이중 오픈 이슈 제거)
- 협정 저장 시 동일 공고 덮어쓰기 동작으로 정리
  - 기준: 발주처(owner) + 업종(industry) + 공고번호(noticeNo), noticeNo 없으면 제목
- 과거 중복 파일 정리용 스크립트 추가
  - `npm run cleanup:agreement-boards` (dry-run)
  - `npm run cleanup:agreement-boards -- --apply` (실삭제)

### C. 협정보드 점수계산 웹 이관(핵심)
- 기존 `electronAPI.formulasEvaluate` 의존 경로를 웹 API 경로로 이관
- formulas/rules 관련 웹 API + 클라이언트 계층 추가
- 협정보드/설정/규칙모달/자동협정/엑셀도우미의 계산 호출을 공통 클라이언트로 연결

## 3) Vercel 배포 이슈 대응

### 이슈 1: Hobby 함수 개수 제한(12개)
- 원인: 분리된 API 파일 개수 초과
- 조치: API 라우트 통합
  - `api/agreement-board/index.js`
  - `api/formulas/index.js`
  - `api/agreements-rules/index.js`
- 현재 함수 파일 수: 7개(제한 이내)

### 이슈 2: Serverless Function 250MB 초과
- 원인: 서버 번들에 `electron` 의존이 끌려들어올 가능성
- 조치: `src/shared/formulas.js`의 정적 `require('electron')` 제거(런타임 require 방식으로 변경)

## 4) 현재 저장 규칙 정리
- 협정 파일은 Blob `company-search/agreement-board/` 경로에 저장
- 파일명(경로 키)에 owner/range/notice 정보 반영
- UI 불러오기 목록은 파일명 직노출이 아니라 메타 기준 표시
- 같은 공고 저장 시 덮어쓰기(신규 중복 생성 방지)

## 5) 추가된 스크립트
- 기존 협정 이관: `npm run migrate:agreement-boards -- <source-directory>`
- 중복 협정 정리:
  - 확인만: `npm run cleanup:agreement-boards`
  - 실제 삭제: `npm run cleanup:agreement-boards -- --apply`

## 6) 다음 세션 권장 작업 순서
1. Vercel 재배포(가능하면 no-cache 재배포 1회)
2. 협정보드 저장/불러오기/덮어쓰기 재검증
3. 협정보드 계산 체감속도 개선
   - debounce + 배치평가 + 캐시 보강
4. 남은 Electron 전용 기능(엑셀 자동화/메일/records/temp 등) 웹 대체 설계

## 7) 참고
- 로컬 빌드(`npm run build`)는 현재 통과 상태
- Vite 경고(`CJS build deprecated`, chunk size warning)는 치명 에러 아님
