# 웹앱 전환 인계 문서 (2026-03-27)

## 1) 오늘 작업의 큰 방향
- Electron/GUI 의존 기능을 웹 전용 구조로 계속 정리
- 원칙:
  - 이 저장소는 웹 전용으로 작업
  - 가능한 기능은 프론트 로컬 처리
  - 공용 저장이 필요한 것만 Blob/API 사용

## 2) 규칙 정리
- 루트 `규칙.md`에 웹 전용 원칙 추가
  - Electron/IPC/preload 호환을 유지하기 위한 새 코드 지양
  - 웹/데스크톱 겸용보다 웹 전용 단순화 우선
- 협정보드 페이지에서는 메뉴바 제거
  - 협정보드 화면 안에서 다른 메뉴로 바로 이동하지 못하게 변경
- 협정보드 페이지 탭 닫기/새로고침 시 브라우저 기본 경고 추가
  - 문구 커스텀은 불가

## 3) formulas / 점수 계산 정리
- `formulas.overrides.json` 경로 제거
- 기준값은 `src/shared/formulas.defaults.json`만 사용
- `formulas` 계산은 서버 API 왕복이 아니라 프론트 로컬 evaluator로 변경
  - 협정보드 경영점수/실적점수/합산 흐름 체감속도 개선
- 지역사찾기 경영 max 표시 버그 수정
  - 웹 이관 과정에서 `15 + 15 = 30`처럼 보이던 오류 제거
  - GUI와 같은 config 기준으로 표시되게 맞춤

## 4) 업체조회 구조 변경
- 업체조회는 Blob 기반 데이터셋을 더 이상 사용하지 않음
- 루트 `db/` 폴더의 원본 xlsx 3개를 빌드 때 JSON으로 자동 변환
  - 전기 / 통신 / 소방
- 스크립트:
  - `scripts/build-static-datasets.js`
  - `npm run build` 전에 자동 실행
- 웹 런타임은 `/public/datasets/*.json` + 브라우저 로컬 캐시 사용
- 결과:
  - 업체조회 때문에 Blob read 발생하지 않음
  - 검색/지역조회 속도 개선
- 추가 수정:
  - 검색 페이지 직전 조회 자동 복원 제거
  - 검색 페이지 초기 `checkFiles/getRegions` 중복 호출 축소
  - `[전체]` 로컬 검색 필터 누락 버그 수정
- 주의:
  - 통신/소방 xlsx는 `comments` 문제로 `sanitizeXlsx`가 필요했음
  - 현재는 전기/통신/소방 모두 공통으로 sanitize 후 파싱

## 5) 협정보드 저장/불러오기 최적화
- 서버 목록 API(`listAgreementBoards`)는 이제 Blob 전체 스캔을 하지 않고 manifest만 사용
- 클라이언트 캐시 추가:
  - `src/shared/agreementBoardCache.js`
  - 목록 메타 캐시
  - 개별 협정 payload 캐시
- 개별 협정은 `path + savedAt` 기준으로 캐시 사용
  - `savedAt` 동일하면 Blob 본문 재조회 안 함
  - `savedAt` 바뀌면 최신본 다시 읽음
- 현재 의미:
  - 불러오기 모달은 캐시 목록을 먼저 보여줌
  - 같은 협정을 다시 열 때 Blob 사용량 감소
  - 다른 사용자가 저장해도 다음 목록 조회 시 최신 메타로 갱신

## 6) 협정보드 엑셀 내보내기
- 템플릿 업로드/삭제 기능 제거
- 프로젝트 내장 템플릿만 사용
- 웹 브라우저에서 직접 엑셀 생성/다운로드하도록 변경
  - Electron `agreements-export-excel` 의존 제거
- 저장 방식:
  - 기본: 새 파일 생성 후 저장/다운로드
  - 지원 브라우저: `showSaveFilePicker` 사용 가능
- 추가 변경:
  - `기존 파일 선택` 버튼으로 사용자가 직접 기존 `.xlsx` 선택 가능
  - 선택한 파일 핸들에 직접 저장
  - 실패 시 조용히 다운로드 폴더 fallback 하지 않고 오류 표시
  - `파일 선택` 버튼은 제거
- 한계:
  - 웹 다운로드 파일 특성상 Excel에서 `편집 사용`이 뜰 수 있음
  - 이건 웹 다운로드/보안 정책 영향이라 제거 보장 어려움

## 7) 메일 기능 웹 전환
- SMTP 프로필:
  - 사용자별 로컬 저장 유지
- 주소록:
  - 공용 Blob JSON 저장으로 변경
  - API:
    - `api/mail-address-book/index.js`
    - `api/_lib/mail-address-book-store.js`
  - 가져오기 후 `저장`을 눌러야 공용 주소록에 반영
  - Blob 실패 시만 `addressBook.seed.json` fallback 사용
  - 예전 `mail:addressBook` 로컬 주소록은 더 이상 사용 안 함
- 실제 메일 발송:
  - 웹 API 추가
    - `api/mail/send-test.js`
    - `api/mail/send-batch.js`
  - 공용 클라이언트:
    - `src/shared/mailClient.js`
  - SMTP 전송은 기존 `src/main/features/mail/smtpService.js` 재사용
- 첨부 처리:
  - JSON base64 전송은 Vercel 413(`FUNCTION_PAYLOAD_TOO_LARGE`) 문제로 폐기
  - 첨부는 먼저 Blob에 임시 업로드 후, 발송 API에는 Blob 경로만 전달
  - API:
    - `api/mail/upload.js`
  - 발송이 끝나면 임시 첨부 Blob는 즉시 삭제
  - MIME 제한 제거 (`.enc` 포함 비표준 확장자 허용)
- 발송 진행 UI:
  - Electron처럼 전체 완료 후 한 번에 반영이 아니라
  - 업체 1건씩 순차 발송/상태 반영하도록 변경
  - 진행 숫자도 `1/N`, `2/N`처럼 즉시 증가
- 웹 제약:
  - 새로고침 후 첨부파일은 이름만 남고 실제 `File` 객체는 복원 불가
  - 다시 첨부해야 실제 메일에 들어감

## 8) 카카오톡 담당자 자동매칭
- 원인:
  - 아직 `window.electronAPI.searchManyCompanies` 경로를 타고 있었음
- 조치:
  - `searchClient.searchManyCompanies()` 추가
  - 카카오 페이지는 이제 공용 searchClient 사용
- 추가 버그 수정:
  - 웹 검색 응답은 `data`인데 `items`만 읽고 있어서 후보가 빈 배열처럼 처리되던 문제 수정
- 현재 의도:
  - GUI 때 `searchMany` 흐름 그대로
  - 실행 경로만 웹 검색 경로로 이관
  - 중복 업체명 선택 모달 로직도 유지

## 9) 협정보드 상태 저장 관련 메모
- 협정보드 draft는 원래 `localStorage`에 저장되도록 되어 있음
- 오늘 한때 라우트 이동 직전 강제 저장을 넣었다가 다시 제거함
  - 이유:
    - 협정보드는 실제 사용 흐름상 새탭/별도 페이지 성격이 강함
    - 다른 화면으로 자주 이동하는 구조가 아님
    - 불필요한 저장 보강보다는 현재 구조 유지가 나음

## 10) 현재 남아 있는 주요 한계 / 주의점
- 협정보드는 여러 공고를 위한 완전한 멀티 세션 탭 구조는 아님
  - 새 탭 메모리 상태가 다른 탭과 자동 공유되지는 않음
- 협정보드 불러오기 캐시는 실시간 동기화가 아니라 `savedAt` 검증형 캐시
- 메일 발송은 웹에서 첨부 업로드 자체는 피할 수 없음
  - 이유: 서버가 사용자 로컬 파일에 직접 접근 불가
- 협정보드 Excel `편집 사용` 문제는 웹 한계로 남아 있음
