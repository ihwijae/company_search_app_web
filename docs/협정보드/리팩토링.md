# 협정보드 리팩토링 메모

## 목적

현재 협정보드 관련 코드가 한 파일에 너무 많이 몰려 있다.

특히 아래 파일이 크다.

- `src/view/features/agreements/components/AgreementBoardWindow.jsx`: 8029줄
- `src/view/features/agreements/components/AgreementCandidateWindow.jsx`: 153줄
- `src/view/features/agreements/context/AgreementBoardContext.jsx`: 609줄
- `src/main/features/agreements/exportExcel.js`: 752줄

이 상태로 기능을 계속 추가하면 아래 문제가 반복될 가능성이 높다.

- 선언 순서 꼬임으로 인한 실행 오류
- 후보/메인 상태 경계 혼선
- UI 수정이 export 로직에 영향
- export 수정이 후보 로직에 영향
- 기능 추가 때 회귀 버그 발생 가능성 증가

리팩토링 목표는 `기능 변경 없이 역할 분리`를 먼저 하는 것이다.

## 현재 파일 역할

### AgreementBoardWindow.jsx

- 협정보드 메인 화면 렌더링
- 헤더 입력값 처리
- 협정테이블 렌더링
- 업체 배치/교체/제거/드래그
- 후보창 상태 및 후보 표시 데이터 계산
- 경영점수/실적/시평액 표시 계산
- 그룹 요약 계산
- 검색, 메모, 기술자점수창, 엑셀 내보내기 모달 처리
- 엑셀 export payload 조립

### AgreementCandidateWindow.jsx

- 후보 보관함 별도 창 UI
- 후보 검색 버튼 및 검색창 UI
- 후보 목록 표 렌더링
- 후보 선택/삭제 버튼 UI

### AgreementBoardContext.jsx

- 협정보드 전역 상태 관리
- 보드 열기/닫기
- candidates/pinned/excluded/groupAssignments 상태 관리
- 검색 결과를 candidate 데이터로 정규화
- 후보/업체 추가 및 제거
- draft persistence 저장/복원
- alwaysInclude 규칙 로드

### exportExcel.js

- 템플릿 엑셀 파일 열기/복사/저장
- 발주처/금액대별 템플릿 좌표 사용
- 협정 업체 데이터 기록
- 후보 데이터 기록
- 셀 스타일 적용

## 왜 한 번에 리팩토링하면 위험한가

협정보드는 상태가 많고 분기 조건이 많다.

- 발주처별 분기
- 금액대별 분기
- 후보/실배치 분기
- 경영/실적/시평액 계산 분기
- 엑셀 템플릿 좌표 분기

이 상태에서 한 번에 크게 나누면 기존 동작이 깨질 위험이 크다.

따라서 리팩토링은 아래 원칙으로 진행해야 한다.

- 한 번에 크게 뜯지 않는다.
- `순수 계산 로직`부터 분리한다.
- UI 구조는 최대한 늦게 건드린다.
- 각 단계마다 빌드 및 수동 확인을 한다.

## 권장 리팩토링 순서

### 1차: 가장 안전한 분리

가장 먼저 할 대상은 `엑셀 export payload 조립`이다.

이유

- UI 렌더를 거의 안 건드린다.
- 현재도 큰 덩어리로 뭉쳐 있다.
- 기능 변경 없이 분리하기 쉽다.

추천 파일

- `src/shared/agreements/agreementExportPayload.js`

이 파일로 분리할 대상

- header payload 생성
- groups payload 생성
- candidates payload 생성

목표

- `AgreementBoardWindow.jsx` 안의 `handleExportExcel` 내부 조립 코드 축소
- export 데이터 구조를 한 곳에서 관리

### 2차: 후보 표시용 selector 분리

추천 파일

- `src/view/features/agreements/utils/agreementCandidateSelectors.js`

이 파일로 분리할 대상

- 후보창 표시 목록 생성
- 후보 경영 경고 계산
- 후보 가능지분 계산
- 후보 표시용 업체명/담당자/지역/배치용 값 계산

목표

- 후보 관련 계산 로직을 UI 파일 밖으로 이동
- 후보 관련 버그 수정 지점을 한 곳으로 모으기

### 3차: 표시 helper 분리

추천 파일

- `src/view/features/agreements/utils/agreementDisplayUtils.js`

이 파일로 분리할 대상

- 업체명 표시 문자열
- 담당자 표시 규칙
- 가능지분 표시 문자열
- 후보/엑셀 이름 셀에 공통으로 들어가는 문자열 규칙

목표

- 이름/배지/엑셀 이름 셀 규칙을 한 곳에서 관리
- 메인 보드와 후보창과 export 간 문자열 규칙 통일

### 4차: 배치/교체/제거 로직 분리

추천 파일

- `src/view/features/agreements/utils/agreementPlacement.js`

이 파일로 분리할 대상

- 업체 배치
- 업체 교체
- 업체 제거
- drag/drop 이동
- 후보에서 보드로 이동

주의

- 이 단계부터는 상태 변경 로직이 많아서 위험도가 올라간다.
- 단계별 확인이 반드시 필요하다.

### 5차: 후보 상태 관리 분리

추천 파일 또는 훅

- `src/view/features/agreements/hooks/useAgreementCandidates.js`

이 파일로 분리할 대상

- 후보 선택 상태
- 후보 검색 상태
- 후보 플래그 처리
- 후보창 전용 액션

주의

- 이 단계가 가장 조심해야 한다.
- 최근 후보 관련 버그 대부분이 이 경계에서 발생했다.

## 이번 세션 기준 결론

지금 당장 전면 리팩토링은 하지 않는다.

이유

- 후보 기능이 막 안정화되는 중이라 회귀 위험이 있다.
- 지금은 기능 유지가 우선이다.

따라서 다음 세션에서는 아래 순서로 시작하는 것이 가장 안전하다.

1. `agreementExportPayload.js` 분리
2. `agreementCandidateSelectors.js` 분리
3. `agreementDisplayUtils.js` 분리

즉, `UI를 건드리지 않는 계산 로직부터 분리`하는 것이 1차 목표다.
