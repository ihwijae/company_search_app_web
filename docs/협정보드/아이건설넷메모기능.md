# 협정보드 아이건설넷 메모 기능 정리

## 목적

협정보드에서 실제로 구성한 협정 내용을 기준으로,
아이건설넷 공고 메모란에 붙여넣을 텍스트를 빠르게 생성하고
누락을 줄이기 위한 기능이다.

기능 이름은 `아이건설넷 메모`이며,
협정보드 상단 툴바 버튼을 누르면 결과를 화면에 별도로 띄우지 않고
즉시 클립보드에 복사한다.

---

## 현재 UI 위치

- 파일: `src/view/features/agreements/components/AgreementBoardWindow.jsx`
- 위치: 협정보드 상단 툴바
- 버튼 라벨: `아이건설넷 메모`
- 배경색: 협정 문자 생성 버튼과 구분되도록 파스텔톤 연두 계열

버튼 클릭 시:

1. 협정보드 현재 상태를 읽음
2. 메모 문자열 생성 함수 호출
3. 결과가 있으면 클립보드에 복사
4. 결과가 없으면 `아이건설넷 메모로 만들 협정 내용이 없습니다.` 안내

---

## 구조 분리

규칙.md 방향에 맞춰,
협정보드 view는 상태를 모아서 기능 함수를 호출하는 역할만 담당한다.

### view 역할

- 파일: `src/view/features/agreements/components/AgreementBoardWindow.jsx`
- 담당:
  - 버튼 클릭 이벤트
  - 협정보드 상태 전달
  - 클립보드 복사
  - 성공/실패 알림 표시

### 메모 생성 기능 역할

- 파일: `src/shared/agreements/calculations/inconMemo.js`
- 담당:
  - 아이건설넷 메모 생성 규칙 처리
  - 대상 업체 판별
  - 상단 금액 라인 생성
  - 본문/하단 구분선 영역 생성
  - 통신+대전 예외 처리

즉, 메모 생성 본체는 `inconMemo.js`가 담당하고,
협정보드는 필요한 데이터만 넘긴다.

## 협정보드에서 넘기는 값

현재 `AgreementBoardWindow`에서 아래 값을 `buildInconMemoText()`에 넘긴다.

- `fileType`
- `dutyRegions`
- `groupAssignments`
- `groupShares`
- `groupApprovals`
- `participantMap`

의미:

- `fileType`: 공종 (`eung`, `tongsin`, `sobang`)
- `dutyRegions`: 의무지역 배열
- `groupAssignments`: 협정테이블 배치 상태
- `groupShares`: 각 슬롯 지분율
- `groupApprovals`: 행별 승인 상태
- `participantMap`: uid -> 실제 업체 데이터 매핑

## 현재 메모 생성 규칙

아이건설넷 메모는 협정보드에 실제 배치된 협정만 기준으로 생성한다.

### 1. 기본 원칙

- 협정보드에 실제 배치된 행만 대상으로 한다.
- 대상 업체가 없는 협정은 메모에 포함하지 않는다.
- 공란 행은 무시한다.
- 승인값이 `취소`인 행은 메모 생성 대상에서 제외한다.
- 승인값이 `알림`인 행은 메모 생성 대상에 포함한다.

주의:

- 협정 문자 생성과 규칙이 다르다.
- 협정 문자 생성은 `알림`, `취소` 둘 다 제외한다.
- 아이건설넷 메모 생성은 `취소`만 제외한다.

### 2. 상단 `금액` 라인 대상 업체

아래 업체가 협정보드에서 대표사로 배치된 경우,
메모 최상단에 `업체명 금액` 형식으로 한 줄씩 추가한다.

- 삼보종합이앤씨㈜
- ㈜김앤드이
- ㈜스타버킷
- (주)포어링크
- ㈜하나전기
- ㈜동우텍
- 일렉파워㈜
- ㈜비에이에너지
- (주)케이너스
- ㈜아이디스

출력 형식:

- `(주)`/`㈜` 등 법인 표시는 제거
- 예: `하나전기 금액`
- 예: `김앤드이 금액`

중요:

- 이 목록은 대표사일 때만 상단 `금액` 라인으로 들어간다.
- 구성사로 들어간 경우에는 `금액` 라인으로 넣지 않는다.

### 3. 본문 상단 영역 대상 업체

아래 업체가 대표사 또는 구성사로 포함된 협정은
본문 상단 영역에 협정 내용을 넣는다.

- 지음이엔아이㈜
- ㈜대흥디씨티
- 아람이엔테크㈜
- ㈜우진일렉트
- ㈜지음쏠라테크
- 에코엠이엔씨㈜

단, 현재 구현상 실제 상단 본문 영역에 들어가는 건 아래 4개 그룹이다.

- 아람이엔테크㈜
- ㈜우진일렉트
- ㈜지음쏠라테크
- 에코엠이엔씨㈜

이유:

- `지음이엔아이㈜`, `㈜대흥디씨티`는 아래 구분선 영역으로 따로 분리하기로 확정됨

출력 형식:

- 단독 100%면: `업체명 단독`
- 공동협정이면:
  - 첫 줄: `대표사명 지분%`
  - 다음 줄부터: `구성사명 지분% [사업자번호]`

예:

```text
아람이엔테크㈜ 67%
㈜대광건영 33% [410-81-74561]
```

### 4. 구분선 아래 영역 대상 업체

아래 업체가 대표사 또는 구성사로 포함된 협정은
본문 하단 구분선 아래에 넣는다.

- 지음이엔아이㈜
- ㈜대흥디씨티
- 삼영플랜트㈜
- ㈜이엘케이
- ㈜엠라이테크
- 영웅개발㈜

출력 형식:

- 상단 본문과 동일
- 단독이면 `업체명 단독`
- 공동이면 여러 줄 블록
- 구분선은 `-----------`

예:

```text
-----------

지음이엔아이 단독

대흥디씨티 단독
```

중요:

- 예전 대화에서 `지음`, `대흥`처럼 축약하지 않기로 확정됨
- 현재는 전부 `(주)`/`㈜`만 제거하고 전체 이름을 사용한다

### 5. 통신 + 대전 의무지역 예외

특이 규칙:

- 공종이 `통신`
- 의무지역에 `대전`이 포함됨

위 두 조건을 동시에 만족하면,
협정보드에 실제 배치가 없어도 메모 상단에 아래 라인을 강제로 추가한다.

- `아이디스 금액`

즉 `㈜아이디스`는 일반적으로는 대표사 배치 시 `금액` 라인 대상이고,
추가 예외로 `통신 + 대전 의무지역`이면 미배치여도 무조건 추가된다.

## 문자열 형식 규칙

### 회사명 표시

- 기본적으로 원본 회사명을 사용
- 단독 표기나 `금액` 라인에서는 `(주)`, `㈜`, `주식회사` 같은 법인 표기만 제거
- 구성 내용 블록에서는 원본 회사명을 유지

예:

- `㈜하나전기` -> `하나전기 금액`
- `지음이엔아이㈜` -> `지음이엔아이 단독`
- 공동협정 블록에서는 `㈜대광건영` 같은 원본 표기를 그대로 둘 수 있음

### 사업자번호 표시

- 구성사(두 번째 줄부터)는 `[000-00-00000]` 형식으로 표시
- 10자리 숫자일 때 하이픈 포맷 적용

### 블록 구분

- 상단 `금액` 라인들 사이: 줄바꿈
- `금액` 영역과 본문 영역 사이: 빈 줄 1개
- 각 협정 블록 사이: 빈 줄 1개
- 하단 구분 영역 시작: `-----------`

## 수정 시 주의사항

### 1. participantMap 참조 순서

이 기능 추가 중 실제로 한 번 에러가 발생했다.

- 에러: `Cannot access 'participantMap' before initialization`

원인:

- `handleGenerateInconMemo`가 `participantMap` 선언보다 위에 있을 때
  React 렌더 중 TDZ 문제가 발생

주의:

- `participantMap`을 쓰는 콜백은 반드시 `participantMap` 선언 아래에 둔다

### 2. 협정 문자 생성 로직과 혼동 금지

`아이건설넷 메모`와 `협정 문자 생성`은 제외 규칙이 다르다.

- 협정 문자 생성:
  - `알림` 제외
  - `취소` 제외
- 아이건설넷 메모:
  - `알림` 포함
  - `취소` 제외

이 부분을 합치거나 공용화할 때 실수하기 쉽다.

### 3. 지음/대흥 축약 금지

초기 논의에서 `지음 단독`, `대흥 단독`으로 축약하려 했지만
사용자 요청으로 취소되었다.

현재 확정:

- `지음이엔아이 단독`
- `대흥디씨티 단독`

이 규칙을 유지해야 한다.

### 4. 화면 표시 패널 없음

초기에는 좌하단 출력 패널을 만들려 했지만 취소되었다.

현재 확정:

- 메모는 화면에 별도 패널로 보여주지 않음
- 버튼 클릭 시 바로 클립보드 복사만 수행

- 대상 업체 목록 추가/삭제
- 상단/하단 분류 기준 변경
- `금액` 대상 예외 조건 추가
- 특정 발주처/공종/의무지역 조합별 예외
- 출력 형식의 줄바꿈 규칙

`src/shared/agreements/calculations/inconMemo.js` 안에서 먼저 처리하고,
view는 상태 전달과 버튼 동작만 유지하는 방향이 안전하다.
