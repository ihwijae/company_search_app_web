# 협정보드 / 실적관리 검색 확장 정리 (2026-03-18)

## 배경

- 실적관리 화면 검색은 기존에 `공사명/발주처` 중심으로만 사용되고 있었음
- 실제 운영 중에는 발주처 정보를 `시공규모 및 비고`에 적는 경우도 있어서
  기존 검색으로는 원하는 실적을 못 찾는 경우가 있었음
- 검색 결과에서 어떤 위치가 검색어에 걸렸는지 한눈에 보이지 않아
  사용자 입장에서 확인이 번거로운 상태였음

## 이번에 바꾼 내용

### 1. 실적관리 검색 대상 확장

- 실적관리 프로젝트 목록 검색이 아래 필드를 함께 보도록 변경
  - `공사명`
  - `발주처`
  - `시공규모 및 비고(scopeNotes)`
  - 기존처럼 `법인명(corporationName)`도 유지
- 저장소 레이어 `recordsRepository.listProjects()`의 keyword 조건에
  `p.scope_notes LIKE ?`를 추가함

### 2. 검색창 문구 정리

- 실적관리 검색 input placeholder를
  `공사명/발주처/시공규모 및 비고 검색`으로 변경
- 사용자가 어떤 필드까지 검색되는지 UI에서 바로 알 수 있게 정리

### 3. 검색어 강조 표시 추가

- 실적관리 결과 카드에서 검색어가 보이는 위치를 강조하도록 변경
- 강조 대상:
  - 법인명
  - 공사명
  - 발주처
  - 시공규모 및 비고
- 강조 방식:
  - `<mark>` 기반 하이라이트
  - 노란 배경색으로 표시

### 4. 비고 HTML 안전 처리 유지

- `시공규모 및 비고`는 Quill HTML을 저장하고 있어서
  기존 `sanitizeHtml()` 흐름은 유지
- sanitize 후 텍스트 노드 단위로 검색어만 감싸는 방식으로 처리해서
  기존 포맷을 최대한 유지하면서 강조되도록 반영

## 영향 범위

- 메인 검색 조건:
  - `src/main/features/records/recordsRepository.js`
- 실적관리 목록 UI / 강조 처리:
  - `src/view/features/records/pages/RecordsPage.jsx`
- 강조 스타일:
  - `src/styles.css`

## 확인한 점

- `npm run build` 실행
- 빌드 정상 통과

## 참고 메모

- `npx eslint`는 현재 저장소의 `eslint.config.js`와 ESLint 패키지 export 문제 때문에 실행되지 않았음
- 이번 작업으로 검색 대상은 확장됐지만, 검색어 강조는 현재 `실적관리 목록 카드`에만 들어감
- 검색 로직은 `LIKE %keyword%` 기반이라 사용자가 입력한 문자열 전체를 기준으로 찾고 강조함

## 다음 세션 확인 포인트

1. 실적관리에서 발주처를 `시공규모 및 비고`에만 적어둔 데이터가 실제로 검색되는지 확인
2. `삼성`, `한국`, `전력` 같은 키워드 검색 시 공사명/발주처/비고 각각에서 강조가 잘 보이는지 확인
3. 비고 안에 줄바꿈, 굵기, 리스트가 있는 HTML에서도 강조 표시가 깨지지 않는지 확인
4. 검색 결과가 많은 경우 체감 성능 저하가 없는지 확인
