# 협정보드 리팩토링 + LH 간이종심제 인계 메모 (2026-03-11)

## 목적

2026-03-11 세션에서 진행한 협정보드 리팩토링과 `LH 100억~300억 간이형종합심사제(간이종심제)` 대응 작업을 다음 세션에서 바로 이어갈 수 있도록 정리한다.

이번 세션 핵심은 아래 4가지다.

- 협정보드 인라인 편집 공통 로직 분리
- 기술자점수 기능 분리
- 후보 업체 점수 재계산 구조 정리
- `LH 100억~300억` 구간 기준/화면/낙찰이력 기능 추가

---

## 이번 세션에서 반영된 내용

### 1. 협정보드 인라인 편집 공통 로직 분리

기존에는 아래 패턴이 `AgreementBoardWindow.jsx` 안에 반복돼 있었다.

- 셀 클릭 시 편집 시작
- input 전환
- `Enter`
- `Escape`
- `blur`
- 편집 종료 후 commit 호출

이를 아래 파일로 분리했다.

- `src/view/features/agreements/hooks/useBoardInlineEditing.js`

현재 구조:

- `useBoardInlineEditing.js`
  - 편집 중 셀 상태
  - 편집 시작/종료
  - `Enter`, `Escape`, `blur` 공통 처리
- `AgreementBoardWindow.jsx`
  - 렌더링과 이벤트 연결만 수행

분리 대상 셀:

- 지분
- 경영점수
- 실적점수
- 시평액
- 품질점수
- 신인도
- 기술자점수

사용자 확인 결과:

- 각 셀 수정 정상
- `Enter`, `Escape`, 포커스 아웃 동작 정상

주의:

- 계산식은 건드리지 않았고 편집 흐름만 분리함

### 3. 후보 업체 점수 재계산 구조 정리

문제 인식:

- 협정보드에 배치된 업체는 현재 기준 재계산 흐름이 비교적 강했음
- 하지만 후보 보관함 업체는 객체 안에 이미 들어 있던 경영점수/실적점수를 재사용할 여지가 있었음
- 발주처/금액대가 바뀌어도 예전 점수를 그대로 볼 가능성이 있었음

이번 세션에서 이를 보정했다.

수정 파일:

- `src/shared/agreements/calculations/managementScore.js`
- `src/shared/agreements/runner/candidateScoreRunner.js`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

핵심 변경:

- 후보 객체의 기존 `managementScore`류 필드를 무조건 신뢰하지 않도록 조정
- 협정보드에서는 현재 `ownerId + rangeId + fileType` 기준 재평가 결과를 우선 사용
- 후보 보관함 업체도 현재 기준으로 다시 계산되도록 강제
- 단, 사용자가 협정보드에서 직접 수정한 값은 유지

현재 의미:

- 협정테이블 배치 업체
- 후보 보관함 업체

둘 다 현재 협정보드 기준으로 재계산되는 방향으로 맞춰짐

### 4. LH 100억~300억 구간 추가

추가 파일/수정 파일:

추가 내용:

- `LH 100억~300억` 금액 구간 추가
- 경영상태:
  - 신용등급만 사용
  - 만점 `8점`
  - 신용평가 없으면 사실상 `0점`
- 실적점수:
  - `실적합계액 / (공사예정금액 × 실적계수) × 11`
  - 만점 `11점`
  - 헤더에 `실적계수` 입력 추가
  - 기본값 `3`
- 품질점수:
  - 업체별 기본 품질값 `88`
  - 지분 반영 합산 후 구간 점수화
  - 현재 기준:
    - `94 이상` → `4.0`
    - `91 이상 94 미만` → `3.9`
    - `88 이상 91 미만` → `3.8`
    - `85 이상 88 미만` → `3.7`
    - `85 미만` → `3.6`
- 지역경제 기여도:
  - 만점 `0.3`
  - 지역업체 참여비율 기반 자동 계산
  - 헤더에 `지역업체 조정계수` 입력 추가
  - 기본값 `1`

중요:

- `낙찰이력 패널티 점수 차감`은 아직 없음
- 현재는 패널티 대상 업체 강조만 들어간 상태

### 5. LH 100억~300억 협정보드 UI/총점 구조 변경

`AgreementBoardWindow.jsx`에서 `isLh100To300` 분기로 처리했다.

이 구간에서만 바뀌는 점:

- `순공사원가`, `A값`, `순공사원가가점` 영역 제거
- 여성기업/장애인기업 `가점` 체크 제거
- `시공경험점수` 제거
- `입찰점수` 제거
- `지역경제 기여도`를 `품질점수` 오른쪽으로 이동
- `기타점수` 컬럼 추가
  - 고정 `17점`
- `예상점수` 계산식 변경

현재 LH 100~300 예상점수 계산식:

- `경영점수 + 실적총점 + 품질총점 + 지역경제기여도 + 기타점수`
- `40점` 초과 시 `40점`으로 캡

추가 강조:

- 지역경제 기여도가 만점 `0.3`이면 초록색 볼드로 강조

주의:

- 이 변경은 `LH 100~300억`에만 적용
- `LH 50억 미만`, `LH 50~100억`, 타 발주처는 유지

### 6. LH 낙찰이력 기능 추가

사용자 요구:

- 공고일 기준 1년 이내 계약 이력이 있으면 패널티 대상 업체로 강조
- 기준 날짜는 `계약일`
- 협정보드와 후보 보관함 모두 업체명을 빨간색 볼드 + `!` 아이콘으로 표시
- 후보 버튼 옆에 `낙찰이력업체` 버튼 추가
- 누르면 별도 창에서 낙찰이력 업체 목록 원문 표시

새 파일:

- `src/shared/agreements/lhAwardHistory.js`
- `src/view/features/agreements/components/LhAwardHistoryWindow.jsx`
- `src/view/features/agreements/hooks/useLhAwardHistoryWindow.js`

역할:

- `lhAwardHistory.js`
  - 낙찰이력 업체 목록
  - 계약일 목록
  - 업체명 정규화
  - 공고일 기준 1년 이내 여부 판정
  - 별도 창에 표시할 원문 텍스트
- `LhAwardHistoryWindow.jsx`
  - 낙찰이력 업체 목록 별도 창 UI
- `useLhAwardHistoryWindow.js`
  - 별도 창 생성/닫기/portal root/style copy/beforeunload 정리

연결 파일:

- `src/shared/agreements/calculations/boardMemberMeta.js`
- `src/shared/agreements/calculations/candidateSelectors.js`
- `src/view/features/agreements/components/AgreementCandidateWindow.jsx`
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`

현재 동작:

- LH `100~300억`에서만 `낙찰이력업체` 버튼 표시
- 공고일 기준 1년 이내 계약 이력이 있는 업체는
  - 협정보드 업체명: 빨간색 볼드 + `!`
  - 후보 보관함 업체명: 빨간색 볼드 + `!`
- 낙찰이력업체 버튼 클릭 시 별도 창 열림

향후 유지보수 포인트:

## 현재 구조 판단

이번 세션에서 합의된 중요한 설계 원칙:

- 협정보드 view에는 가능한 한 새 규칙을 직접 넣지 않음
- 데이터/계산/판정은 `shared`
- 창 관리와 상태 흐름은 `hooks`
- 화면 표시만 `components`

즉 앞으로도 새 규칙이나 기능이 생기면 아래 방향으로 가는 것이 맞음.

- 계산식 변경
  - `shared/agreements/calculations/*`
- 별도 데이터셋
  - `shared/agreements/*`
- 외부 창/워크플로우
  - `view/features/agreements/hooks/*`
- view
  - 연결과 렌더링만 유지
