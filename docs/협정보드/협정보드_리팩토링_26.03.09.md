# 협정보드 리팩토링 인계 메모 (2026-03-09)

## 목적

협정보드 관련 리팩토링 진행 상태를 다음 세션에 바로 이어갈 수 있도록 현재 구조, 완료된 분리, 제거된 기능, 남은 우선순위를 정리한다.

이번 문서는 다음 내용을 기준으로 업데이트한 인계본이다.

- 기존 계산 로직 분리 상태 반영
- `복사 기능` 제거 반영
- 다음 세션부터 실제로 건드릴 남은 덩어리 우선순위 재정리

## 현재 상태 요약

현재 협정보드는 `계산 본체`와 `상태 변경 본체` 상당 부분이 이미 `shared` 계층으로 빠진 상태다.

구조 기준:

- `src/view/features/agreements/...`

  - React state

  - 이벤트 연결
  - 외부 창/모달 제어

- `src/shared/agreements/calculations/...`
  - 경영/실적/시평/가능지분/summary 계산
  - 후보 표시 selector
  - 슬롯 표시 meta

- `src/shared/agreements/runner/...`
  - 후보 점수 재계산 실행 흐름

- `src/shared/agreements/placement/...`
  - 협정테이블 배치/교체/제거/드롭 상태 변경 본체

  - 엑셀 실제 쓰기

현재 주요 파일 크기:

- `src/view/features/agreements/components/AgreementBoardWindow.jsx`: 6669줄
- `src/view/features/agreements/context/AgreementBoardContext.jsx`: 609줄

중요:

- 이전보다 많이 줄었지만 `AgreementBoardWindow.jsx`는 여전히 크다.
- 지금 남은 덩어리는 대부분 `계산 로직`이 아니라 `view 오케스트레이션`이다.

## 이번 세션에서 반영된 내용

### 1. 복사 기능 제거

협정보드의 `복사` 기능은 더 이상 사용하지 않는 것으로 보고 제거했다.

제거된 것:

- 툴바의 `복사` 버튼
- 복사 모달
- 업체명/지분/경영/실적/시평액 복사 로직
- 관련 상태
- 관련 핸들러
- 미사용 dead code `copyGroupMetric`

중요:

- `협정 문자 생성` 기능은 절대 삭제하지 않았음
- `협정 문자 생성`은 현재도 유지 대상이며 정상 기능으로 간주함

### 2. 회귀 이슈 1건 정리

복사 기능 제거 과정에서 `formatShareDecimal` 함수까지 같이 제거되어 런타임 에러가 발생했었다.

에러:

- `ReferenceError: formatShareDecimal is not defined`

원인:

- 이 함수가 복사 전용이 아니라 협정보드 지분 셀 표시에도 사용되고 있었음

조치:

- 공용 지분 표시 함수 `formatShareDecimal`만 복구
- 복사 기능은 복구하지 않음

즉:

- 복사 기능 제거 상태는 유지됨
- 공용 표시 함수만 되돌린 상태임

## 다음 세션에서 헷갈리면 안 되는 원칙

### 1. 계산 규칙 추가는 view가 아니라 shared에 넣는다

예:

- 새 발주처 추가
- 새 금액대 추가
- 금액대별 경영상태 기준 변경
- 실적 산식 변경
- 시평액 반영 기준 변경
- summary 총점 규칙 변경

이런 건 `AgreementBoardWindow.jsx`에 조건문을 넣지 말고 아래 계층에 넣는다.

- `src/shared/agreements/calculations/...`
- 필요 시 `src/shared/agreements/runner/...`
- 필요 시 `src/shared/agreements/placement/...`
- 필요 시 `src/shared/agreements/agreementExportPayload.js`
- 필요 시 `src/main/features/agreements/exportExcel.js`

`AgreementBoardWindow.jsx`는 선택 UI, 표시, 이벤트 연결만 담당하도록 유지하는 것이 목표다.

### 2. `협정 문자 생성`은 복사 기능이 아니다

이번 세션에서 제거한 것은 `복사 모달 기반 데이터 복사 기능`이다.

유지 대상:

- `협정 문자 생성`

제거 대상:

- `복사` 버튼
- 복사 모달
- 데이터 복사용 클립보드 출력 로직

다음 세션에서도 이 둘을 섞어서 다루면 안 된다.

## 추천 우선순위

복사 기능 제거 반영 후, 다음 세션 우선순위는 아래 순서가 가장 적절하다.

1. `보드 검색 분리`
2. `인라인 편집 공통 로직 분리`
3. `기술자 점수 워크플로우 분리`
4. `팝업/외부 창 관리 공통화`
5. `보드 셀/행 렌더링 컴포넌트 분리`
6. `헤더 입력 상태/연결 정리`

## 다음 세션 시작 시 권장 방식

다음 세션에서 바로 구현을 시작할 때는 이렇게 요청하는 것이 안전하다.

예시:

- `협정보드 보드 검색 기능을 AgreementBoardWindow.jsx 밖으로 분리해줘. 기존 동작은 유지하고 view에는 연결만 남겨줘.`
- `새 금액대 규칙을 추가해줘. 계산 규칙은 shared/agreements 쪽에 구현하고 AgreementBoardWindow.jsx에는 계산 분기를 넣지 말아줘.`

이렇게 명시하면 다시 view에 계산 로직이 몰리는 것을 줄일 수 있다.

## 인계용 한 줄 요약

협정보드는 계산/배치 본체가 상당 부분 `shared`로 빠진 상태이며, 이번 세션에서 사용하지 않는 `복사 기능`은 제거했다. 다음 세션 1순위는 `보드 검색 분리`, 그다음은 `인라인 편집 공통화`, `기술자 점수 워크플로우 분리`다.
