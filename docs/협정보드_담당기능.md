# 협정보드 담당 기능 정리

## 목적

협정보드 관련 파일이 많아지면서,

- 어떤 파일이 무엇을 담당하는지
- 어디를 수정해야 하는지
- 계산/표시/UI/실행 흐름이 어떻게 나뉘는지

를 빠르게 확인할 수 있도록 현재 구조를 정리한다.

---

## 큰 구조

### 1. `view`

역할:
- 화면 렌더링
- 이벤트 연결
- confirm/alert 호출
- React state 관리

대표 파일:
- `src/view/features/agreements/components/AgreementBoardWindow.jsx`
- `src/view/features/agreements/components/AgreementCandidateWindow.jsx`
- `src/view/features/agreements/components/CandidatesModal.jsx`

---

### 2. `shared/calculations`

역할:
- 공용 계산 로직
- 값 추출 로직
- 표시용 selector/meta/helper
- 점수 계산 본체

---

### 3. `shared/runner`

역할:
- 계산을 실제로 실행하는 흐름
- 재계산 필요 여부 판단
- 계산 결과 반영

---

### 4. `shared/placement`

역할:
- 협정테이블 업체 배치/교체/제거 관련 상태 변경 로직

---

### 5. `main/features/agreements`

역할:
- Electron main process
- 엑셀 실제 작성
- 파일 처리

---

## 현재 주요 파일별 담당 기능

### `src/view/features/agreements/components/AgreementBoardWindow.jsx`

현재 담당:
- 협정보드 메인 화면 렌더링
- 헤더 입력/버튼/모달 UI
- confirm/alert 호출
- React state 연결
- 후보창 열기/닫기
- 보드 검색, 복사, 편집 등 아직 남은 UI 흐름
- shared 로직 호출 결과를 화면에 반영

현재 성격:
- UI 조립 중심 파일
- 예전보다 계산/조립/배치 본체가 많이 빠진 상태

---

### `src/view/features/agreements/components/AgreementCandidateWindow.jsx`

담당:
- 후보 보관함 별도 창 UI
- 후보 표 렌더링
- 후보 검색 버튼/검색창 UI
- 후보 선택/삭제 버튼 UI
- 드래그 시작 UI 이벤트

중요:
- 계산은 하지 않음
- 메인에서 만든 entries를 받아서 보여주는 역할

---

### `src/view/features/agreements/components/CandidatesModal.jsx`

담당:
- 지역사찾기(후보 찾기) 화면 UI
- 검색 결과/정렬/필터/선택 UI
- 부채/유동/신용 breakdown 표시
- 단독가능, 여성기업, 품질 등 지역사찾기 전용 표시

현재 반영:
- 경영점수 총점 계산은 공용 `managementScore.js` 사용
- 지역사찾기 화면 전용 breakdown UI는 그대로 유지

---

### `src/shared/agreements/agreementExportPayload.js`

담당:
- 엑셀 내보내기용 payload 조립
- `header`
- `groups`
- `candidates`

중요:
- 실제 엑셀 파일에 쓰는 것은 아님
- 내보내기용 데이터 객체를 만드는 역할

---

## calculations 폴더

### `src/shared/agreements/calculations/possibleShare.js`

담당:
- 가능지분 비율 계산
- 가능지분 숫자 포맷
- 가능지분 텍스트 포맷

사용처:
- 협정테이블
- 후보
- 엑셀 payload

---

### `src/shared/agreements/calculations/candidateSelectors.js`

담당:
- 후보창에 보여줄 entries 생성
- 후보 검색용 텍스트 조합
- 후보 경영 경고 여부
- 후보 가능지분 표시값
- 후보 업체명/지역/담당자 표시값 조립

사용처:
- 후보 보관함

---

### `src/shared/agreements/calculations/boardMemberMeta.js`

담당:
- 협정테이블 한 칸에 필요한 표시용 meta 생성

포함 값:
- 업체명
- 담당자
- 가능지분
- 경영 경고
- 실적/시평 표시값
- 자료상태
- 태그
- 비고

사용처:
- 협정테이블 렌더링

---

### `src/shared/agreements/calculations/displayText.js`

담당:
- 표시 문자열 조합

현재 포함:
- `buildBoardNameCellText()`

사용처:
- 협정테이블 업체명 복사

---

### `src/shared/agreements/calculations/groupSummary.js`

담당:
- 그룹 단위 summary 계산

포함 기능:
- 지분합/지분 유효성
- 가중 경영점수
- 가중 실적값
- 가중 기술자점수
- 시평액 합/가중 시평액
- 입찰참가가능 시평 조건
- 총점/최대점 계산

사용처:
- 협정테이블 summary 칸

---

### `src/shared/agreements/calculations/managementScore.js`

담당:
- 경영점수 계산 본체
- 신용등급 추출
- 신용 만료 판단

포함 기능:
- 수동 입력 우선 처리
- 만점 플래그 처리
- 캐시 사용
- 직접 필드/합산 필드 해석
- 부채/유동/신용 점수 해석

사용처:
- 협정테이블
- 후보
- 그룹 summary
- 지역사찾기 총점 계산

---

### `src/shared/agreements/calculations/performanceValue.js`

담당:
- 실적 값 선택/추출

포함 기능:
- 기본 실적값 추출
- 현재 금액대 기준 실적값 선택
- `행안부 50억~100억` 구간에서 `3년 실적` 사용

사용처:
- 협정테이블
- 후보
- 그룹 summary
- 후보 재계산 runner

---

### `src/shared/agreements/calculations/sipyungValue.js`

담당:
- 시평액 값 추출
- 시평액 캐시 처리

사용처:
- 협정테이블
- 후보
- 그룹 summary
- 복사 기능

---

### `src/shared/agreements/calculations/performanceScore.js`

담당:
- 실적점수 계산 본체
- performance cap 해석
- formulasEvaluate 응답 해석
- fallback 점수 계산

중요:
- 실적 “값 선택” 파일과는 별개
- 이 파일은 실적 “점수 계산” 담당

---

## runner 폴더

### `src/shared/agreements/runner/candidateScoreRunner.js`

담당:
- 후보/업체 점수 재계산 운영 흐름

포함 기능:
- 어떤 업체를 재계산할지 판단
- cache key 생성/캐시 상태 처리
- formulasEvaluate 호출
- 경영점수/실적점수 결과를 candidate에 반영
- 갱신된 후보 수 반환

중요:
- 계산식 자체는 아님
- 계산을 “언제 누구에게 실행할지” 담당

---

## placement 폴더

### `src/shared/agreements/placement/boardPlacement.js`

담당:
- 협정테이블 배치/교체/제거 관련 상태 변경 로직
- 이동 관련 일부 분기/문구 조합

포함 기능:
- 슬롯에 업체 배치
- 첫 빈칸 배치
- 교체/이동
- 드래그 드롭 assignment 변경
- 슬롯 비우기
- 이동 대상 분기 판단
- 이동/배치 문구 조합

중요:
- confirm 호출, alert 호출, React state setter는 담당하지 않음
- 상태를 어떻게 바꿀지만 담당

---

## main 쪽 파일

### `src/main/features/agreements/exportExcel.js`

담당:
- 실제 엑셀 파일 작성
- 템플릿 시트 열기/복사/저장
- 셀 값 쓰기
- 셀 스타일/색상 적용
- 후보 업체 내보내기

중요:
- 프론트 payload를 받아 실제 엑셀에 쓰는 역할

## 인계용 요약

현재 협정보드는

- 핵심 계산 로직
- 후보 점수 재계산 흐름
- 협정테이블 배치 상태 변경 로직

이 상당 부분 `shared` 쪽으로 분리된 상태다.

다음 세션에서 기능 수정 시 우선 파일을 이 문서 기준으로 찾으면 된다.
