# 협정보드 후보 경영점수 및 다른 정보 정리

## 개요

협정보드의 `후보 보관함`은 별도의 화면처럼 보이지만, 실제 데이터는 협정보드 메인 상태를 공유한다.  
즉 후보창이 따로 업체 데이터를 보관하거나 따로 점수를 계산하는 구조가 아니다.

핵심 원칙은 다음과 같다.

- `협정보드 메인`이 전체 후보 목록(`candidates`)을 가진다.
- `협정 테이블`은 그중 실제 배치된 업체만 보여준다.
- `후보 보관함`은 전체 후보 목록 중 `협정 테이블에 배치되지 않은 업체`만 보여준다.


## 후보와 메인의 관계

사용자가 후보창에서 업체를 검색해서 추가해도, 그 업체는 후보창 전용 저장소에만 들어가는 것이 아니다.  
실제로는 협정보드 메인의 `candidates` 상태에 추가된다.

즉 구조는 아래와 같다.

1. 후보창에서 업체 검색
2. 업체 선택
3. 협정보드 메인의 `candidates`에 추가
4. 후보창은 그 `candidates` 중 미배치 업체만 표시

정리하면:

- 화면상으로는 `후보에만 있는 업체`
- 실제 상태상으로는 `협정보드 메인이 이미 가지고 있는 업체`


## 경영점수는 어디서 계산되는가

경영점수는 `후보창에서 계산하지 않는다`.  
경영점수 계산 주체는 협정보드 메인이다.

후보창은 메인에서 계산된 결과를 받아서 보여주는 역할이다.

즉:

- `경영점수 계산`: 메인
- `후보창`: 표시


## 5년실적, 시평액도 같은가

기본 개념은 같다.

- `5년실적`, `시평액`도 메인 상태 기준으로 관리된다.
- 후보창은 그 값을 보여주는 역할이다.

다만 차이는 있다.

- `5년실적`, `시평액`은 원본 데이터에서 바로 읽히는 경우가 많다.
- `경영점수`는 계산이 더 들어가는 경우가 많다.

그래도 구조적으로는 모두 메인 상태를 기준으로 후보창에 표시된다.


## 후보에만 있을 때도 점수가 나오는 이유

후보에만 있어 보여도, 실제로는 메인 상태의 `candidates` 안에 이미 들어 있기 때문이다.

그래서 메인 쪽 계산 로직은:

- 협정 테이블에 올라간 업체만 보는 것이 아니라
- 메인이 가진 전체 후보 목록도 계산 대상으로 볼 수 있다.

즉:

- 후보에만 있어도 계산 대상이 될 수 있다.
- 협정 테이블에 올라가지 않아도 경영점수 계산은 가능하다.


## 왜 예전에 후보에서 경영점수가 `-`로 보였는가

원인은 `후보창이 비동기 계산 완료 후의 갱신을 다시 반영하지 못한 것`이다.

흐름은 이랬다.

1. 후보에 업체 추가
2. 메인에서 경영점수 계산 시작
3. 계산은 완료됨
4. 하지만 후보 리스트가 그 갱신 신호를 못 보고 예전 값을 계속 들고 있음
5. 그래서 후보창에는 `-` 표시
6. 나중에 다른 상태 변화가 생기면 다시 렌더되면서 점수가 보임

즉 계산이 안 된 게 아니라, `후보 화면 반영 타이밍` 문제였다.


## 수정된 방향

후보 리스트가 메인의 계산 결과 갱신을 다시 보도록 수정했다.

의미상으로는 아래와 같다.

- 메인에서 경영점수 계산
- 계산 완료 후 내부 버전값 갱신
- 후보 리스트가 그 버전값 변화를 감지
- 후보창도 다시 렌더
- 최신 경영점수 표시

쉽게 말하면:

`후보창이 협정보드 메인에 있는 최신 점수를 다시 읽어서 보여주도록 맞춘 것`


## 전화번호 제거와 경영점수 문제의 관계

사용자가 `전화번호는 DB에 없다`고 확인한 뒤 전화번호 표시를 제거했다.

이 변경 자체는 화면 표시용 필드 제거에 가깝고, 경영점수 계산 로직과 직접 연결되지는 않는다.

따라서:

- `전화번호 제거 자체가 원인`은 아님
- 다만 그 전후로 후보용 데이터 구성과 화면 렌더링을 여러 번 수정하는 과정에서
  후보 화면 갱신 타이밍 문제가 함께 드러난 것에 가깝다.


## 후보창에서 검색할 때 메인 화면을 안 타는 것 아닌가

UI는 후보창에서 검색하지만, 데이터는 결국 메인 상태로 들어간다.

즉:

- 검색 모달은 후보창 위에서 뜰 수 있음
- 하지만 선택한 업체는 메인의 `candidates`로 들어감
- 메인이 그 데이터를 기준으로 점수를 계산함
- 후보창은 그 결과를 보여줌

따라서 `후보창에서 검색해서 추가했기 때문에 경영점수 계산이 안 된다`는 구조는 아니다.


## 현재 구조 요약

- 메인이 전체 후보 데이터를 가진다.
- 협정 테이블은 배치된 업체만 보여준다.
- 후보창은 미배치 업체만 보여준다.
- 경영점수 계산은 메인에서 한다.
- 후보창은 메인 계산 결과를 표시한다.
- 5년실적, 시평액도 메인 기준 값을 표시한다.


## 실무적으로 이해하면 좋은 문장

가장 쉽게 말하면 아래 문장으로 정리할 수 있다.

`후보에만 있어 보여도 실제로는 협정보드 메인이 이미 가지고 있는 업체이고, 후보창은 메인에서 계산된 점수를 보여주는 화면이다.`
